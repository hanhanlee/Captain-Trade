## 第一章：這套工具能幫你做什麼

### 工具定位

> **這套工具是「輔助決策系統」，不是自動下單機器人。**
>
> 它的角色是：幫你看、幫你算、幫你提醒——但買不買、賣不賣，最終仍由你決定。

### 五大核心問題，工具給你答案

| 你的問題 | 工具的回答 | 對應功能 |
|----------|-----------|---------|
| 今天哪些股票值得關注？ | 列出技術面符合條件的候選清單，依強度排名 | 選股雷達 |
| 我手上的股票該不該賣？ | 即時計算損益，自動偵測 5 種危險訊號，推播 LINE | 持股監控 |
| 這次應該買幾張？ | 根據你的帳戶資金與停損位置，科學計算建議張數 | 風險控制 |
| 現在大盤環境好不好？ | 用 3 個訊號判斷多頭 / 偏多 / 偏空 / 空頭 | 市場環境 |
| 我的整體表現如何？ | 統計真實勝率、盈虧比、情緒對決策的影響 | 交易日誌 |

## 第二章：設計哲學與核心理念

### 以傳奇操盤手的眼界設計

本工具的設計原則借鑒了以下幾位市場傳奇人物的核心思想：

**Paul Tudor Jones（保羅·都鐸·瓊斯）**
- 「最重要的是防守，而非進攻。」
- 從他身上借鑒：嚴格的停損紀律、帳戶最大回撤監控

**Stan Druckenmiller（斯坦利·德魯肯米勒）**
- 「市場環境比選股更重要。在錯誤的環境進場，再好的股票也容易虧損。」
- 從他身上借鑒：進場前必先判斷大盤多空環境

**Ed Seykota（艾德·塞科塔）**
- 「長期成功的關鍵是：截斷虧損、讓利潤奔跑、嚴格控制部位大小。」
- 從他身上借鑒：固定風險法的部位計算邏輯

**William O'Neil（威廉·歐尼爾）**
- 「量能放大的突破，才是真正有效的買進訊號。」
- 從他身上借鑒：量能確認的篩選條件

### 核心設計原則

```
第一優先：保住本金
第二優先：控制風險
第三優先：尋找獲利機會
```

這個順序非常重要。多數投資人把順序倒過來，先想著賺多少，忘了先想萬一虧損怎麼辦。

## 第四章：如何啟動工具

### 建議啟動方式：Srock CLI

新版提供 `srock` 命令列管理工具，用來統一啟動本機 Dashboard、Basic Auth 反向代理與 Tailscale Funnel。第一次使用若系統還找不到 `srock` 指令，請先在專案資料夾執行一次：

```
cd "E:\workspace\srock tool"
python -m pip install -e .
```

之後可用以下指令操作：

| 指令 | 用途 |
|------|------|
| `srock status` | 查看 Streamlit、Auth Proxy、Funnel 目前狀態 |
| `srock up --profile local` | 只啟動本機 Streamlit（開發或只在自己電腦使用） |
| `srock up --profile protected` | 啟動 Streamlit + Caddy Basic Auth，但不開外網 |
| `srock up --profile full` | 啟動 Streamlit + Caddy Basic Auth + Tailscale Funnel，對外提供受保護入口 |
| `srock down` | 依序關閉 Funnel、Auth Proxy、Streamlit |
| `srock restart --profile protected` | 重啟受保護的本機入口 |
| `srock watch` | 即時監控服務狀態 |
| `srock logs all` | 查看 Streamlit / Caddy 最近 log |
| `srock open` | 開啟本機 Streamlit |
| `srock open --public` | 開啟目前 Funnel 對外網址 |

如果你只想快速啟動預設設定，也可以雙擊專案根目錄的 `srock-up.bat`。它會進入專案資料夾並執行 `srock up`。

### 服務啟動完成通知

使用 `srock up` 或 `srock restart` 啟動服務時，系統會等 Streamlit、Auth Proxy 與 Cloudflare Tunnel 依照 profile 啟動完成後，推播一則 LINE 訊息給所有啟用中的訂閱者。若你分開手動啟動元件，最後執行 `srock start funnel` 並取得 Cloudflare 對外網址後，也會送出同一種啟動通知。訊息內容包含：

- 服務已啟動
- 啟動完成時間
- 目前可用網址；`full` profile 會優先顯示 Cloudflare Tunnel 對外網址，`protected` / `local` 則顯示本機網址

為避免半夜或清晨打擾，啟動通知只會在本機時間 **08:00 到 20:00** 之間送出。時間外啟動服務仍會正常啟動，只是不推播啟動訊息。這個通知只掛在 CLI 啟動流程，不會因為 Streamlit 頁面重新整理而重複推播。

## 第五章：什麼是技術分析

### 基本定義

技術分析是一種透過**研究過去的價格和成交量**，來預測未來價格走勢的方法。

它的核心假設有三點：

**假設一：價格反映一切（Market Discounts Everything）**
所有影響股價的資訊——財報、新聞、法人動態——最終都會反映在價格上。因此，研究價格本身就已經包含了所有資訊。

**假設二：價格按趨勢移動（Price Moves in Trends）**
價格一旦形成趨勢，傾向延續，直到出現明確的反轉訊號。這是動能策略的理論根基。

**假設三：歷史會重演（History Repeats Itself）**
人類的恐懼和貪婪是不變的，過去出現的價格型態，在類似的心理環境下會再次出現。

### 技術分析 vs. 基本分析

| 面向 | 技術分析 | 基本分析 |
|------|---------|---------|
| 看什麼 | 價格圖、成交量、指標 | 財報、獲利、競爭優勢 |
| 時間框架 | 較短期（數天到數月） | 較長期（數月到數年） |
| 核心問題 | 「現在的趨勢方向是什麼？」 | 「這家公司值多少錢？」 |
| 缺點 | 不考慮公司實際價值 | 無法判斷進場時機 |
| 適合 | 波段操作、判斷進出場時機 | 長期持有、選擇好公司 |

> 本工具以技術分析為主，適合**波段操作**的使用情境。長期存股的投資者更應該以基本分析為主。

## 第六章：各指標的學術根據與可靠性

以下逐一說明本工具使用的每個指標，包含它的理論來源和實際可靠程度。

---

### 指標一：移動平均線（Moving Average，MA）

**發明背景**

移動平均線的概念起源於道氏理論（Dow Theory，1900 年代初期），是技術分析中歷史最悠久的工具。

**原理**

MA20 = 最近 20 個交易日收盤價的算術平均值

這個數值代表「市場近期的平均持倉成本」。當現價高於均線，代表目前持有者平均是獲利的，市場情緒偏多；當現價低於均線，代表多數人是虧損的，市場情緒偏空。

**本工具的具體應用**

- **條件：** 收盤價 > MA20，且今日 MA20 > 5 日前的 MA20
- **意義：** 不只是「站上均線」，還要求「均線本身向上」，確認趨勢在持續而非只是短暫反彈

**學術證據**

Brock, Lakonishok & LeBaron（1992，Journal of Finance）對道瓊指數近百年的數據進行研究，發現均線交叉策略在統計上具有顯著的預測能力，超過隨機預期。

**可靠程度：★★★★☆**

均線是最廣泛使用的指標，在趨勢型行情中效果穩定。主要缺點是在橫盤震盪時會發出大量假訊號（上下穿越卻沒有趨勢延續）。

---

### 指標二：成交量（Volume）

**原理**

成交量代表在特定時間內，市場上實際發生的買賣交易量。量能放大表示更多參與者在此價格帶交易，方向的確定性更高。

**本工具的具體應用**

- **條件：** 今日成交量 > 最近 5 日平均量 × 1.3
- **意義：** 量比至少 1.3，代表今天的交投比平常活躍 30% 以上，確認這次的價格移動有實質買盤支撐

量能對價格的意義：

| 量價關係 | 解讀 |
|---------|------|
| 量增價漲 | 多方強勢，趨勢確認，最理想 |
| 量縮價漲 | 買盤不足，漲勢恐難持續 |
| 量增價跌 | 空方強勢，可能加速下跌 |
| 量縮價跌 | 賣壓不強，可能即將止跌 |

**學術證據**

Chordia, Subrahmanyam & Anshuman（2001，Journal of Financial Economics）證明成交量與未來報酬的相關性：高量能的價格突破，後續延續的機率顯著高於低量能突破。

Lee & Swaminathan（2000，Journal of Finance）研究發現，高成交量動能股票的未來表現顯著優於低成交量動能股票。

**可靠程度：★★★★★**

量能是技術分析中獲得最多學術支持的指標之一，各流派幾乎都將量能視為確認訊號的必要條件。

---

### 指標三：MACD（指數平滑移動平均線）

**發明背景**

由 Gerald Appel 於 1970 年代發明，是目前全球使用最廣泛的技術指標之一。

**原理**

MACD 由三個元素組成：

```
快線（DIF）= 12日指數移動平均 - 26日指數移動平均
慢線（DEA）= DIF 的 9日指數移動平均
柱狀圖    = (DIF - DEA) × 2
```

指數移動平均（EMA）與一般均線的差別在於，越近期的價格權重越高，對最新的市場變化更敏感。

**黃金交叉的意義**

當 DIF 從下方穿越 DEA（由負轉正）：
- 代表短期動能超越了長期動能
- 市場的買方力量正在增強
- 是趨勢翻多的早期訊號

**本工具的具體應用**

滿足以下任一條件即計分：
1. DIF 剛剛從下穿越 DEA（黃金交叉當天）
2. DIF 在零線以上且 DIF > DEA（動能已在多方，持續強勢）

**MACD 的本質限制**

MACD 是「落後指標」（Lagging Indicator）。它確認已發生的趨勢，不預測未來。這意味著收到訊號時，股票往往已經漲了一段距離。因此本工具將 MACD 作為確認條件，而非唯一買進理由。

**可靠程度：★★★☆☆**

效果因市場環境而異。趨勢明確時可靠；橫盤時假訊號多。需配合其他指標使用。

---

### 指標四：RSI（相對強弱指數）

**發明背景**

由 J. Welles Wilder 於 1978 年在著作《Technical Analysis of New Concepts》中發表。

**原理**

```
RSI = 100 - [100 ÷ (1 + 平均漲幅 ÷ 平均跌幅)]
```

RSI 衡量的是：在過去 N 天（本工具使用 14 天）中，漲幅占總波動的比例。數值介於 0 到 100 之間。

**傳統解讀 vs. 本工具的用法**

| RSI 數值 | 傳統解讀 | 本工具的解讀 |
|---------|---------|------------|
| 0 - 30 | 超賣，可能反彈 | 不入選（股票可能持續弱勢） |
| 30 - 50 | 偏弱 | 不入選 |
| **50 - 70** | **強勢健康區** | **入選（健康強勢，還有空間）** |
| 70 以上 | 超買，可能回調 | 不入選（已漲過頭，風險偏高） |

**為什麼選 50-70 而不是「超賣反彈」？**

傳統用法是在超賣（RSI < 30）時買進，等待反彈。但本工具採用相反的邏輯——選強勢股，而不是選弱勢反彈。

這個選擇來自動能投資的研究：真正的強勢股不會長期在超賣區間，它會在 50-70 的健康區間穩步前進。超賣股雖然可能反彈，但背後往往有基本面問題，反彈後繼續下跌的機率也很高。

**可靠程度：★★★☆☆**

RSI 的閾值設定（50-70）是一種選擇，不同市場、不同個股的最佳區間可能不同。作為過濾器使用效果較好，單獨使用則較不可靠。

---

### 指標五：布林通道（Bollinger Bands）

**發明背景**

由 John Bollinger 於 1980 年代發展，以統計學的標準差概念為基礎。

**原理**

```
中軌 = MA20（20日移動平均）
上軌 = MA20 + 2 × 標準差
下軌 = MA20 - 2 × 標準差
```

統計上，正常情況下約 95% 的價格應該落在上下軌之間。

**本工具的具體應用**

- **條件：** 收盤價必須高於布林通道下軌
- **意義：** 跌破下軌代表跌幅已超出統計上的「正常波動範圍」，是弱勢股的特徵，直接排除不考慮

這是一個**排除條件**，而非買進理由——用來篩掉明顯弱勢的股票。

**可靠程度：★★★★☆**

作為排除過濾器使用非常穩定。布林通道下軌突破通常代表賣壓大於正常水準，是客觀的統計事實。

---

### 籌碼面加分：法人買超與融資動向

**法人買超（外資、投信）**

三大法人擁有龐大的研究資源和資金，他們的買賣動作通常基於深度的基本面分析。法人持續買超的股票，代表「聰明錢」認同這個價位。

注意：法人也會做錯，但統計上，法人買超的股票表現優於法人賣超的股票。

**融資餘額減少**

融資是散戶以槓桿借錢買股。融資過多的股票有一個風險：當股價下跌，融資戶被迫斷頭賣出，形成「殺多」的連鎖反應，加速下跌。

融資減少（籌碼乾淨）代表：
- 持股者多為自有資金，不容易被迫賣出
- 較不受到非理性的恐慌賣壓影響

## 第七章：技術分析的限制：誠實的評估

### 這套策略可靠嗎？完整的答案

**可靠，但只在特定條件下有優勢，而且優勢是機率性的，不是確定性的。**

---

### 支持技術分析的學術證據

| 研究 | 發表期刊 | 結論 |
|------|---------|------|
| Jegadeesh & Titman (1993) | Journal of Finance | 動能效應真實存在：過去 3-12 個月強勢的股票，未來 3-12 個月繼續強勢的機率高於市場平均 |
| Brock, Lakonishok & LeBaron (1992) | Journal of Finance | 均線策略在道瓊指數百年數據中具有統計顯著性 |
| Fama & French (後續研究) | 多處 | 雖為效率市場假說支持者，仍承認動能是「無法用傳統模型解釋的異常報酬」 |
| Lo, Mamaysky & Wang (2000) | Journal of Finance | 某些技術型態在統計上具有預測價值 |

---

### 反對技術分析的論點

**1. 效率市場假說（Efficient Market Hypothesis，EMH）**

Eugene Fama（1970，諾貝爾經濟學獎得主）的理論主張：

> 「股票價格已充分反映所有公開資訊，因此不可能透過分析公開資訊（包括歷史價格）來持續跑贏市場。」

這個理論與技術分析直接對立。學術界對此至今仍有爭議，兩派各有有力證據。

**2. 指標本質上是落後的**

所有技術指標都基於過去的價格。它們能告訴你「過去的趨勢發生了什麼」，但無法預測「未來會發生什麼」。訊號出現時，最好的進場時機往往已經過了一部分。

**3. 自我毀滅效應（Self-defeating Prophecy）**

當越來越多人使用相同的指標和規則，訊號的效果會被提前消化，甚至被主力利用。例如，所有人都知道「突破均線買進」，主力可以故意拉高穿越均線引誘散戶進場，然後立刻出貨。

**4. 台股的特殊風險**

台股法人比例相對成熟市場低，散戶比例高，具有以下特性：
- 消息面、政治事件的影響比成熟市場更大
- 部分個股存在主力操控的風險
- 在主力出貨的過程中，技術圖形可能完全正常，直到突然崩跌

---

### 正確認識這套工具的能力邊界

**一個誠實的比喻：**

> 這套工具就像在賭場中，幫你找到了一張**勝率 55% 的牌桌**（而非標準的 50%）。
>
> 長期下來，你會因為這個微小的優勢而獲利。但任何單一一局仍然可能輸，而且連輸幾局也是正常的。
>
> **真正讓你長期賺錢的，不是找到這張牌桌，而是：**
> - **嚴格執行停損**（輸了要知道離開）
> - **科學計算押注大小**（不能因為「感覺很強」就全押）
> - **只在勝率偏高時進場**（不在偏空環境做多）

### 掃描模式與歷史回放

新版 Sidebar 多了四個實用功能：

1. **掃描模式預設**
- `⚡ 極速`：20 檔、最少額外條件，適合快速確認
- `🎯 標準`：100 檔、開啟法人條件，適合日常使用
- `🔬 完整`：全市場、可搭配融資與族群偵測，適合收盤後完整檢查

2. **分析基準日**
- 可選今天，或回放過去某一天
- 選過去日期時，系統會自動改成**歷史回溯模式**
- 歷史回溯模式只使用本機價格資料，不混入今天才知道的法人、融資或基本面資料

3. **三線齊穿模式**
- `嚴謹型`：昨日收盤必須在 MA5 / MA10 / MA20 三條線下方
- `寬鬆型`：昨日只要仍在三線中的任一線下方即可
- 嚴謹型標的較少但更乾淨；寬鬆型標的較多但雜訊也較高

4. **後續盤勢**
- 歷史回溯掃描完成後，頁面會直接列出 `+1 / +3 / +5 / +10` 日報酬
- 也會計算各期間的最高漲幅，方便快速檢查當天訊號後續表現

### v4 規則總覽

目前選股雷達使用的是 **v4 領先攻擊版**。核心思想不再是「已經走很久的強勢股」，而是找出：

> **均線剛糾結完成，今天才剛啟動突破的股票。**

#### ✅ 必要條件：七項全過才入選

| 條件 | 分數 | 實際判斷 |
|------|------|----------|
| 第一天站上 5/10/20MA | 35 | 今日收盤高於 MA5 / MA10 / MA20，且昨日仍在三線下方（依嚴謹/寬鬆模式判定） |
| 均線糾結度 < 3%（昨日）| 20 | `(max(MA5,MA10,MA20) - min(...)) / MA20`，**以昨日均線計算**，確認突破前處於盤整 |
| 量能爆發比 > 1.5 倍 | 15 | 今日成交量 > 5 日均量 × 1.5 |
| ATR 過熱防護通過 | 10 | 今日收盤未超過 `MA20 + 3.5 × ATR14`（倍數可在 Sidebar 調整） |
| 相對強度 RS > 80 | 10 | 過去 63 個交易日報酬率明顯跑贏大盤，RS 分數 > 80 |
| 突破 60 日收盤新高 | 10 | 今日收盤 > 過去 60 個交易日的最高收盤，確認為真實突破 |
| 主力連續買超 3 日以上 | 10 | 讀取 `broker_main_force_cache`，最近 3 個交易日「前 15 大買超分點 - 前 15 大賣超分點」皆為正 |

七項全部成立，基礎分就是 **110 分**。只要其中一項沒過，就不會進入候選名單。

> **均線糾結度為何改查昨日？** 突破當天的漲幅往往會直接撐開均線距離，若用今日數值反而會把最強的爆量突破股篩掉。改用昨日狀態，才能正確確認「盤整後才突破」。

> **為什麼主力連 3 日買超是必要條件？** v4 找的是「剛啟動且有資金確認」的股票。若只有價格突破、沒有主力資金連續承接，訊號容易變成假突破，因此目前主力連續買超 3 日以上已列為必要條件。這裡的「主力」不是三大法人合計，而是 Sponsor 分點券商資料計算出的主力買賣超；若 `broker_main_force_cache` 尚未補齊，這項會判定未達。

#### ➕ 加分條件：用來排序強弱

| 條件 | 分數 | 意義 |
|------|------|------|
| 布林頻寬縮減 | 10 | 今天的布林頻寬低於 20 天前，代表波動收斂後準備擴張 |
| 投信第一天買超 | 10 | 昨日投信不為正，今日轉正，表示新資金剛進場 |
| 週線 MA10 扣抵值低位 | 10 | 10 週前收盤低於當前週 MA10，代表週線均線未來有自然上揚條件 |
| 融資減少 / 籌碼集中 | 5 | 融資餘額較前一日下降，代表散戶去槓桿 |

**滿分 = 必要條件 110 + 加分 35 = 145 分。**

### v3 / v4 策略差異

選股雷達可以在 Sidebar 切換 **v4 領先攻擊版**與 **v3 均線突破版**。兩者不是同一套策略：

| 策略 | 核心用途 | 必要條件 |
|------|----------|----------|
| v4 領先攻擊版 | 找「盤整剛結束、今日啟動突破」的股票 | 三線齊穿、均線糾結、量能爆發、ATR 過熱防護、RS > 80、60 日新高、主力連 3 日買超 |
| v3 均線突破版 | 找「已站上趨勢、技術條件轉強」的股票 | 站上 MA20、MA20 向上、量增、不低於布林下軌、MACD 或 RSI 轉強、主力連 3 日買超 |

v4 比 v3 嚴格很多，因此常見狀況是：

- v3 有候選股，但 v4 沒有候選股
- v4 候選股數量很少，但條件更集中在「剛突破」與「資金確認」
- 兩者都會要求主力連續買超 3 日以上，因此 Sponsor 分點主力快取尚未更新時，兩個策略都可能沒有結果

結果表的「觸發訊號」會依策略版本分開顯示：選 v3 時只列 v3 相關訊號；選 v4 時只列 v4 相關訊號。這避免 V3 結果混入 V4 標籤，造成兩個策略看起來一樣。

### 過熱股防護機制（ATR 動態門檻）

選股雷達雖然找啟動股，但仍會防止追得太高。新版改用 **ATR 動態門檻**，取代舊版固定的月線乖離率（%）：

```
過熱條件：今日收盤 > MA20 + N × ATR14
```

ATR（真實波動幅度）會自動隨個股波動特性縮放——波動大的半導體股門檻自然更寬；波動小的傳產股門檻自然更嚴，不會用同一把尺誤殺熱門領漲股。

Sidebar 可以設定：

1. **過熱 ATR 倍數**（預設 3.5）
   - 一般多頭環境維持 3.5
   - 半導體或熱門電子股噴發期可調至 4.5 ～ 5.0
   - 0：停用 ATR 過熱防護

2. **超過門檻時的處置**
- **直接剔除**：保守型做法
- **扣分懲罰**：保留觀察，但讓排名往後

---

### 分數怎麼看

| 強度分數 | 解讀 |
|---------|------|
| 110 分 | 剛好通過七項必要條件，屬於「成立但尚未特別強」 |
| 115-125 分 | 必要條件外，有一到兩項加分確認 |
| 130 分以上 | 必要條件完整，且多項加分同步成立，屬精選強勢候選 |

分數不是保證報酬，而是幫你把「形態最乾淨、資金確認較多」的股票排到前面。

### 產業族群分析（第二個頁籤）

除了個股排行榜，選股雷達還提供「產業族群分析」功能，幫你看懂資金流向。

**什麼是產業族群分析？**

當選股雷達同時選出 3 檔散熱模組股票、2 檔 AI 伺服器股票，這代表資金正在集中流入這兩個產業。這種現象叫做「族群輪動」——機構法人通常以「產業佈局」的方式進場，而不是隨機選股。

**如何解讀？**

| 顯示資訊 | 意義 |
|---------|------|
| 各產業入選檔數 | 入選越多，代表這個產業正在吸引更多資金 |
| 產業平均分數 | 分數越高，代表這個產業的整體技術面越強 |
| 各產業最強股 | 在最熱門的產業中，分數最高的就是最值得研究的標的 |

**操作建議：**

1. 找出入選最多的 1-2 個產業
2. 在那個產業中，選分數最高的 1-2 檔股票深入研究
3. 「最強產業裡的最強股票」是機構法人的基本操作邏輯

---

### 流動性前置過濾（掃描前先縮小範圍）

在開始掃描前，可以先用流動性條件過濾股票宇宙，聚焦真正值得關注的標的：

| 模式 | 說明 | 建議使用時機 |
|------|------|------------|
| **前日量前 N 名**（預設 100）| 鎖定昨天成交最活躍的 100 檔股票 | 日常使用，動態追蹤市場熱點 |
| **日均量 ≥ N 張**（預設 1000，上限 50,000）| 近 20 日平均量超過 N 張才考慮 | 想篩掉低流動性股票時使用 |
| 不過濾 | 掃描全部股票 | 週末深度掃描 |

> **為什麼用量能代替資本額？** 日均量超過 1000 張的股票，資本額幾乎必然超過 20 億，兩個條件高度重疊。而量能是每日動態更新的，比靜態的資本額更能反映當下市場關注度。

### 產業輪動過濾（只掃最強族群）

在流動性過濾之上，還可以再加一層「產業輪動過濾」：

**原理：** 計算所有已下載股票的近 5 日（一週）漲幅，按產業分組取平均，選出漲幅排名前 N 個產業，只在這些產業內做技術篩選。

**操作：** Sidebar 勾選「只掃描近一週漲幅前 N 類股」，預設取前 3 個產業。

**掃描完成後頁面會顯示：**
```
📊 本次鎖定產業（近 5 日漲幅前 3 名）：
  半導體 (+4.2%)　散熱模組 (+3.8%)　AI伺服器 (+3.1%)
```

**三層過濾疊加效果：**
```
全市場 950 檔
  → 前日量前 100 名（流動性過濾）
  → 近一週漲幅前 3 個產業（產業輪動過濾）
  → 技術分析篩選（必要條件全過）
  → 最終候選：5-15 檔精選個股
```

這種由上而下（Top-Down）的篩選邏輯，和機構法人的選股流程一致：**先選大環境 → 再選產業 → 最後選個股**。

### 族群集體突破偵測（🔥 新功能）

在 Sidebar「族群集體突破偵測」區塊可啟用兩個進階的族群動量指標，掃描後在「產業族群」頁籤的最底部顯示結果。

#### 指標一：族群創高密度（HP Density）

**原理：** 計算族群內有多少比例的股票今日收盤等於近 N 日最高收盤。

```
族群 A 有 20 檔股票
其中 8 檔今天收盤創了 20 日新高
→ HP Density = 8 ÷ 20 = 40%
```

**意義：** 一個族群裡有 40% 以上的股票同時創高，代表這個族群是「集體突破」，而非個別股票的偶然創高。這種族群性的動作通常代表機構法人正在整族群佈局。

**可調整參數：**
- 創高天數 N（預設 20 日）
- 族群創高比例門檻（預設 30%）

#### 指標二：資金流向比重（Turnover Ratio）

**原理：** 以「收盤價 × 成交量」估算各族群的成交額，計算其佔全市場的比重，找出資金最集中的族群。

```
族群 B 今日估算成交額 = 80 億
全市場估算成交額 = 2000 億
→ 資金比重 = 80 ÷ 2000 = 4%（排名第 2）
```

**意義：** 錢往哪裡流，股票就往哪裡漲。資金比重排名前 N 的族群，代表今天市場最積極追逐的方向。

**可調整參數：**
- 資金前幾大族群（預設 5）

#### 集體突破判斷邏輯

| 啟用指標 | 判斷方式 |
|---------|---------|
| 只啟用 HP Density | 創高比例 ≥ 門檻 → 🔥 集體突破 |
| 只啟用 Turnover | 成交額排名 ≤ N → 🔥 集體突破 |
| 兩個都啟用 | 兩個條件同時達到 → 🔥 集體突破 |

### 籌碼面條件：法人買超（可自訂模式）

三大法人條件現在支援兩種模式，在 Sidebar 的「納入三大法人買賣超」區塊操作：

> **注意：** 這個開關只處理外資、投信、自營商買賣超，主要用於投信第一天買超、法人買超加分或額外過濾。v3 / v4 的必要條件「主力連 3 日買超」已改用 Sponsor 分點券商資料，不會因為勾選三大法人而自動取得；請先在「資料管理」補完 Sponsor 主力券商快取。

**模式 A：個別法人皆須買超（嚴格模式）**

可自由勾選外資、投信、自營商，系統會檢查：

- 勾選的法人中，**每一個**都必須在**連續 N 個交易日**為淨買超
- 任何一方賣超，或連續天數不足，都不算達標

這個模式適合你想找「多方資金共識高」的股票。

**模式 B：三大法人合計買超（寬鬆模式）**

這個模式不要求三方各自都買超，而是看整體資金是否淨流入。可選兩種判斷方式：

- **近 N 日累計 > 0**：最近 N 個交易日的三大法人合計淨買超總和大於 0
- **連續 N 日每日 > 0**：最近 N 個交易日中，每一天的三大法人合計淨買超都大於 0

這種模式可減少自營商避險單對訊號的干擾，更適合觀察整體資金方向。

**法人加分規則**

| 模式 | 觸發條件 | 加分 |
|------|---------|------|
| 個別法人皆須買超 | 勾選法人各自連續買超 | +7 |
| 三大法人合計買超 | 合計淨買超達標 | +4 |
| 三大法人合計買超 + 外資投信同步買超 | 合計達標，且外資與投信同時連續買超 | +7（= 4 + 3） |

**必要條件 vs. 加分條件**

- **加分條件（預設）**：符合法人條件才加分，不符合不影響入選
- **必要條件**：勾選「法人條件為必要條件」後，不符合的股票直接剔除

建議：
- 想找「有明確法人背書」的強勢股 → 用「個別法人皆須買超」並設為必要條件
- 想找「整體資金在流入」的股票 → 用「三大法人合計買超」

```
第一步：先看「市場環境」頁面，確認大盤現在是多頭還是空頭
第二步：空頭環境 → 選股雷達結果僅供參考，不建議進場
         多頭環境 → 進行以下步驟
第三步：在 Sidebar 設定掃描條件：
         - 流動性過濾：建議選「前日量前100名」
         - 產業輪動：建議勾選「前3個產業」
         - 有法人資料需求時，依風格選擇「個別法人皆須買超」或「三大法人合計買超」
         - 若不想追價過熱股，可開啟「ATR 過熱防護」，預設上限為 MA20 + 3.5 × ATR14
第四步：執行掃描，找出分數 65 分以上的股票
第五步：看「產業族群」頁籤，確認哪個產業最強
第六步：在最強產業中，點入分數最高的個股查看 K 線圖，確認：

         - 突破盤整了嗎？量能是否集中在上漲日？
         - RS 分數是否 > 70（是否在市場中屬於強勢股）？
第七步：找到合理的進場點和停損位置
第八步：前往「風險控制」計算應該買幾張
第九步：進場後在「交易日誌」新增 BUY 紀錄，系統會自動同步到持股監控
```

| 欄位 | 是否必填 | 說明 |

### 持股圖表與布林帶寬

持股監控的個股圖表現在會同時畫出：

- 收盤價走勢
- MA5 / MA10 / MA20
- 布林通道上軌 / 中軌 / 下軌
- 布林通道填色區域
- 布林帶寬百分比

布林帶寬用來衡量目前波動區間有多寬：

```
布林帶寬% = (布林上軌 - 布林下軌) ÷ 布林中軌 × 100%
```

帶寬變窄代表波動收斂，後續可能進入方向選擇；帶寬突然擴大代表波動放大，若同時跌破重要均線或停損價，風險會升高。

### 停損價如何設定？

這是最重要的欄位。以下是幾種常見的設定方法：

**方法一：固定比例法（最簡單）**
- 設定成本價的 8-10% 以下為停損
- 例如：成本 100 元 → 停損設在 90-92 元

**方法二：技術支撐位（較精準）**
- 在 K 線圖上找到最近的支撐位（前低、均線、整數關卡）
- 停損設在支撐位下方一點點
- 例如：前低在 95 元 → 停損設在 93 元

**方法三：布林通道下軌**
- 以當前布林通道下軌作為停損參考
- 跌破下軌代表已超出正常波動範圍

> **原則：停損要在進場前就設好，不能在股票跌了之後才想。**

### 五種警示規則完整說明

#### 🔴 危險警示（建議立即檢視，盡快決定）

**危險一：跌破停損價**

- 觸發條件：現價 ≤ 你設定的停損價
- 意義：這是你事先設定的「計畫底線」，跌破代表這筆交易已超出你的風險承受範圍
- 建議行動：依照你的計畫執行停損

停損的數學邏輯：

| 虧損幅度 | 需要的回本漲幅 |
|---------|-------------|
| -5% | +5.3% |
| -10% | +11.1% |
| -20% | +25.0% |
| -30% | +42.9% |
| -50% | +100.0% |

虧損越大，需要的回本幅度呈指數級增加。及早停損是數學上的必然選擇。

**危險二：從近期高點回撤超過 12%**

- 觸發條件：現價比近 60 日最高點跌超過 12%
- 意義：即使沒有設停損，這個幅度代表之前的帳面獲利已大量侵蝕，趨勢可能反轉
- 建議行動：重新評估是否繼續持有，考慮縮減部位

---

#### 🟡 注意警示（需要關注，提前準備應對）

**注意三：從高點回撤 8-12%**

- 觸發條件：回撤幅度在 8% 到 12% 之間
- 意義：開始出現明顯的回撤，尚未到危險程度但需要提高警覺
- 建議行動：收緊停損設定，或考慮先賣出一部分

**注意四：跌破 20 日均線**

- 觸發條件：今日收盤價 < MA20
- 意義：短期趨勢轉弱，買進的理由之一（站上均線）已不再成立
- 建議行動：重新評估持有理由是否仍然存在

**注意五：無停損設定且虧損超過 -5%**

- 觸發條件：沒有填入停損價，且損益% < -5%
- 意義：沒有保護機制的虧損正在擴大
- 建議行動：立刻設定停損價，不要讓虧損繼續無限擴大

---

#### 🔵 提示（好消息，考慮採取行動）

**提示：達到停利目標**

- 觸發條件：現價 ≥ 你設定的停利價
- 意義：計畫中的獲利目標已達到
- 建議行動：考慮全部或部分獲利了結，或上移停損鎖住部分利潤

## 第十章：風險控制 — 每次應該買幾張

### 為什麼需要科學計算部位大小

> 「很多投資人失敗，不是因為選錯股，而是押注太大。一筆重押的虧損，足以把好幾筆獲利全部抵銷。」

假設你有 10 筆交易，9 筆各賺 1 萬元，1 筆重押虧損 10 萬元：
- 總結果 = 9 × 10,000 - 100,000 = -10,000 元（虧損！）

如果每筆都用科學的方式控制風險，讓每筆最多虧 2 萬元：
- 總結果 = 9 × 10,000 - 20,000 = +70,000 元（獲利！）

部位大小的控制，是讓勝率優勢真正轉化為獲利的關鍵。

---

### 計算方法一：固定風險法（日常使用首選）

**核心原則：每筆交易，最多虧損帳戶總資金的 2%。**

為什麼是 2%？
- 1%：太保守，即使連續虧損也不會傷到帳戶，但獲利能力也受限
- 2%：業界常見標準，連續虧損 10 筆才損失帳戶的 18%，帳戶仍存活
- 5% 以上：太激進，連續虧損 5-6 筆可能損失帳戶的 25% 以上

**計算公式：**

```
最大風險金額 = 帳戶總資金 × 2%
每股風險金額 = 進場價 - 停損價
可買股數     = 最大風險金額 ÷ 每股風險金額
建議張數     = 無條件捨去（可買股數 ÷ 1000）
```

**實際範例：**

```
帳戶：1,000,000 元
最大風險：2% = 20,000 元

情境 A（停損偏寬）：
  進場價 850 元，停損 780 元（每股風險 70 元）
  可買 = 20,000 ÷ 70 = 285 股 = 0.28 張 → 建議 0 張（此交易不划算）

情境 B（停損合理）：
  進場價 850 元，停損 820 元（每股風險 30 元）
  可買 = 20,000 ÷ 30 = 666 股 = 0.66 張 → 建議 1 張

情境 C（低價股）：
  進場價 50 元，停損 46 元（每股風險 4 元）
  可買 = 20,000 ÷ 4 = 5,000 股 = 5 張 → 建議 5 張
```

**重要認知：停損越緊 = 可以買越多張；停損越寬 = 只能買越少**

這看似矛盾，但邏輯是對的：停損緊代表你接受的每股損失小，相同的總風險預算可以買更多股。

---

### 計算方法二：Kelly Criterion（有歷史記錄後使用）

Kelly 公式由數學家 John Larry Kelly Jr. 於 1956 年發表，最初用於電話訊號的傳輸，後來被賭博界和投資界廣泛採用。

**公式：**

```
Kelly% = 勝率 - (1 - 勝率) ÷ 盈虧比

其中：
  勝率    = 過去交易中獲利次數 ÷ 總次數
  盈虧比  = 平均獲利金額 ÷ 平均虧損金額
```

**範例計算：**

假設你有 20 筆交易記錄：
```
勝率    = 11/20 = 55%
平均獲利 = 20,000 元
平均虧損 = 10,000 元
盈虧比   = 20,000 / 10,000 = 2

Kelly% = 0.55 - (0.45 ÷ 2) = 0.55 - 0.225 = 32.5%

意義：每次交易應使用帳戶的 32.5%
```

**為什麼本工具只用 25% 的 Kelly 值（Fractional Kelly）？**

Full Kelly 在理論上是「最大化長期資產增長率」的最佳解，但有兩個實際問題：

1. **你的歷史統計可能不準確**：樣本數不夠多時，估算的勝率和盈虧比誤差很大
2. **波動太大，心理難以承受**：Full Kelly 可能要求你把三分之一的帳戶押在一筆交易上

使用 25% Fractional Kelly（即 0.25 × 32.5% = 8.1%）可以：
- 大幅降低帳戶波動
- 對估算誤差更具容忍度
- 長期複利效果仍然良好

> **建議：** 剛開始使用時先用「固定風險法」，累積至少 30 筆交易記錄後，再嘗試 Kelly Criterion。

### 計算方法三：ATR 正金字塔加碼計畫（趨勢持續時使用）

很多人加碼的方式是錯的：股價一跌就想攤平，股價一漲又一次重壓。這樣做會讓風險越來越失控。

新版「正金字塔加碼計算機」改成 **ATR 動態版**。它不再用固定的 50% / 30% / 20% 資金比例和 BIAS 上限，而是用進場當時的波動度（Initial ATR）當作共同尺度，自動算出加碼觸發價、加碼張數、初始停損、熔斷器與移動停利線。

#### 模組一：初始設定

你需要輸入：

| 欄位 | 說明 |
|------|------|
| 進場價 | 第一筆部位的實際進場價 |
| 進場時 ATR(14) | 進場當時的 14 日 ATR，用來衡量初始波動 |
| 基礎部位（張） | 第一筆買進幾張 |

系統會先計算初始停損：

```
初始停損 = 進場價 - 2 × Initial ATR
```

#### 模組二：動態階梯加碼

| 層級 | 觸發價 | 加碼張數 |
|------|--------|----------|
| Level 1 | 進場價 + 0.5 × Initial ATR | 基礎部位 ÷ 2，至少 1 張 |
| Level 2 | Level 1 觸發價 + 1.0 × Initial ATR | 基礎部位 ÷ 4，至少 1 張 |

這代表你不是固定在「獲利 5%」或「獲利 15%」加碼，而是依照該股票自己的波動特性來判斷。波動大的股票需要走更遠才算真正突破；波動小的股票則不需要硬等固定百分比。

#### 模組三：雙重過熱熔斷器

只要任一熔斷條件成立，系統就會阻擋加碼，即使價格已經到達 Level 1 或 Level 2。

| 熔斷條件 | 判斷方式 | 意義 |
|----------|----------|------|
| ATR 膨脹 | Current ATR > Initial ATR × 1.5 | 波動突然失控，追價風險升高 |
| 布林上軌超買 | 目前股價 > 布林上軌 | 股價短線衝出統計高檔區 |
| RSI(9) 超買 | RSI(9) > 80 | 短線動能過熱，容易震盪 |

布林上軌與 RSI(9) 是選填欄位；不填時只使用 ATR 膨脹檢查。

#### 模組四：ATR 移動停利

系統同時會用目前 ATR 計算移動停利線：

```
移動停利線 = 進場以來最高收盤 - 2 × Current ATR
```

並顯示目前已鎖定的獲利比例：

```
已鎖定獲利% = (移動停利線 - 進場價) ÷ 進場價 × 100%
```

**使用建議：**

1. 第一筆仍先用固定風險法決定初始部位
2. 進場後若股價往有利方向移動，再打開「正金字塔加碼計算機」
3. 只在 Level 1 / Level 2 觸發且熔斷器未啟動時加碼
4. 若移動停利線已接近現價，優先思考保護獲利，而不是繼續加碼

> **提醒：** 正金字塔加碼法的前提是「你已經做對方向」。如果第一筆還沒獲利，就不應進入第二、第三階段。

### 報酬風險比（R:R）— 每次進場前的必算指標

**公式：**

```
報酬風險比 = (目標價 - 進場價) ÷ (進場價 - 停損價)
```

**為什麼這個數字這麼重要？**

用數學說明：假設你的勝率固定在 40%（每 10 筆只贏 4 筆）

```
情境 A：RR = 1（賺虧一樣多，例如賺 10,000 / 虧 10,000）
  10 筆總結果 = 4 × 10,000 - 6 × 10,000 = -20,000 元（虧損）

情境 B：RR = 2（賺的是虧的 2 倍，例如賺 20,000 / 虧 10,000）
  10 筆總結果 = 4 × 20,000 - 6 × 10,000 = +20,000 元（獲利！）

情境 C：RR = 3（賺 30,000 / 虧 10,000）
  10 筆總結果 = 4 × 30,000 - 6 × 10,000 = +60,000 元（大幅獲利）
```

即使只有四成勝率，只要 RR ≥ 2，長期下來就能獲利。

**RR 值的行動建議：**

| RR 值 | 評估 | 建議 |
|-------|------|------|
| < 1 | 不合理 | 絕對不進場，每次都賺少虧多 |
| 1 - 1.5 | 偏低 | 需要很高的勝率才能獲利，謹慎 |
| 1.5 - 2 | 尚可 | 配合選股雷達的訊號才進場 |
| **2 - 3** | **理想** | **積極評估進場機會** |
| > 3 | 優秀 | 這種機會不常見，出現時要把握 |

### 方法三：ATR 移動停利（持倉中使用）

**從「固定停利」進化到「動態停利」**

傳統的固定停利（例如：漲 15% 就賣）有一個缺點：遇到真正的大行情時，你會太早被賣出去，只賺到小段，錯失後面的大漲。

ATR 移動停利解決了這個問題——它隨著股價上漲而「往上移動停損線」，讓你在噴出大行情時能守住更多獲利，同時又能防止回吐過多。

**什麼是 ATR（平均真實振幅）？**

ATR = 最近 14 天的平均每日波動範圍

- 波動大的股票，ATR 數值大（例如台積電 ATR 可能是 15-20 元）
- 波動小的股票，ATR 數值小（低波動股可能只有 2-3 元）

ATR 讓停損線能「自適應」個股的波動特性，不會因為股票本來就震盪大而輕易被洗出場。

**移動停損線的計算公式：**

```
移動停損線 = 持有期間最高價 − N × ATR（N 通常設為 2）

例如：
  持有期間最高股價：120 元
  ATR（14日）：   3 元
  N = 2 倍

  移動停損線 = 120 − 2 × 3 = 114 元

  → 只要收盤沒有跌破 114 元，繼續持有
  → 如果股票繼續漲到 130 元，停損線自動上移至 124 元
```

**三種狀態說明：**

| 系統顯示 | 意義 | 建議動作 |
|---------|------|---------|
| ✅ 安全 | 股價與停損線之間有充足緩衝 | 繼續持有，讓獲利奔跑 |
| ⚠️ 接近停損 | 股價與停損線的距離不足 2% | 密切注意，準備應對 |
| 🔴 已觸發出場 | 股價已跌破移動停損線 | 依紀律執行出場 |

**如何在工具中使用：**

前往「風險控制」→「部位計算」→ 展開「ATR 移動停利計算器」：
1. 輸入你的進場成本
2. 輸入目前股價
3. 輸入持有期間最高股價（自己查看 K 線圖）
4. 輸入 ATR 值（從 K 線圖技術指標讀取，或以近期每日波動幅度估算）
5. 設定 ATR 倍數（建議從 2 倍開始；高波動股用 2.5-3 倍）

> **建議：** 進場時先設定固定停損（依 8-10% 或技術支撐位），股價上漲獲利超過 10% 之後，再切換到 ATR 移動停利來保護獲利。

除了單筆交易的風險，系統也監控你整個帳戶的健康狀況：

### 產業集中度曝險（進階風控）

**為什麼單看「每筆 2% 風險」還不夠？**

假設你持有 5 檔股票，每筆都嚴格控制在 2% 風險，看起來很安全。
但如果這 5 檔全部都是「AI 伺服器概念股」，問題就來了：

一旦出現以下任何一種狀況——
- 大型 AI 公司公布資本支出削減
- 台積電法說會下修展望
- 美國政府對 AI 晶片出口新增限制

你的 5 檔持股會「同時」觸發停損，在同一天損失帳戶的 10%。

**這就是「相關性風險」**：每筆個別風險是 2%，但因為高度相關，整體風險遠超過你以為的。

**本工具的解法：**

在「帳戶曝險」頁籤中，點擊「分析產業曝險」按鈕：
- 系統自動從 FinMind 取得每檔持股的產業分類
- 計算每個產業的合計市值佔帳戶比例
- 超過設定上限（預設 7%，可自行調整）即顯示警示

**示意圖（假設情境）：**

```
你的持股：                    產業歸屬：       佔帳戶比：
  2330 台積電（15張）   →   半導體         12.3%  ⚠️ 超過 7%
  2379 瑞昱（3張）      →   半導體          2.1%
  2454 聯發科（2張）    →   半導體          3.8%
  ─────────────────────────────────────────────
  半導體合計                                18.2%  🔴 嚴重超標

系統警示：「產業『半導體』佔帳戶 18.2%，同產業利空將同時觸發停損」
```

**如何降低產業集中風險？**

1. 把持股分散在不同產業（例如：半導體、金融、消費、工業）
2. 如果你特別看好某個產業，至少確保同產業合計不超過 7-8%
3. 最強產業中只選 1-2 檔，而非同時持有該產業的所有強勢股

---

**保留現金的重要性：**

保留 20% 以上的現金有三個作用：
1. 出現絕佳機會時，還有子彈可以進場
2. 市場急跌時，不需要被迫賤賣持股換現金
3. 心理上更穩定，不會因為全倉而過度緊張

## 第十一章：市場環境 — 積極還是保守

### 為什麼要先看大盤

個股再好，在空頭環境下也容易被大盤拖累。研究顯示，在多頭環境中，約 70% 的股票會跟著上漲；在空頭環境中，約 70% 的股票會跟著下跌。

先判斷大盤環境，再選股，才能站在機率有利的一側。

### 三個判斷訊號

| 訊號 | 判斷條件 | 代表意義 |
|------|---------|---------|
| 短期趨勢 | 加權指數 > MA20 | 近一個月市場偏多頭 |
| 中期趨勢 | 加權指數 > MA60 | 近三個月市場偏多頭 |
| 動能方向 | 大盤 MACD DIF > DEA | 市場整體動能偏多方 |

### 四種環境與對應策略

**🟢 多頭（3 個訊號都滿足）**

市場環境最佳。多數股票都在上漲，選股的成功率最高。

建議策略：
- 積極使用選股雷達尋找機會
- 每筆可使用標準風險（2% 帳戶）
- 可同時持有 3-5 檔股票
- 設定合理停損，讓獲利奔跑

**🟡 偏多震盪（2 個訊號滿足）**

市場偏多但不明確，可能即將轉強或轉弱。

建議策略：
- 只買分數最高（前 3 名）的強勢股
- 適度縮小部位（每筆 1-1.5% 帳戶風險）
- 持股數量控制在 2-3 檔以內
- 停損要設得比平常更緊

**🟠 偏空震盪（1 個訊號滿足）**

市場已偏弱，進場的賠率不佳。

建議策略：
- 不主動買進新倉位
- 守好現有持股，收緊停損
- 把精力放在研究，等待環境改善
- 可考慮縮減部位、增加現金比例

**🔴 空頭（0 個訊號）**

市場明確下跌，多數股票都在跌。

建議策略：
- 停止任何做多操作
- 能出場的持股考慮出場
- 持有現金等待
- 空頭結束、環境轉多時再進場

### 一個重要提醒

市場環境判斷是大方向的參考，不是 100% 精準的預測。偶爾會出現大盤空頭但個股強勢的情況，或大盤多頭但個股弱勢的情況。環境判斷是機率的考量，不是鐵律。

## 第十二章：交易日誌 — 記錄與學習

### 為什麼記錄比選股更重要

傳奇操盤手 Paul Tudor Jones 說：「不分析自己的交易，等於在黑暗中射箭。」

不記錄的投資人，會不斷重複同樣的錯誤而渾然不知。記錄的目的不是懲罰自己，而是找到規律：
- 哪類交易你最擅長？
- 什麼情緒下你最容易做錯誤決定？
- 你的真實勝率和盈虧比是多少？

### 每筆交易需要記錄的內容

| 欄位 | 必填 | 說明 |
|------|------|------|
| 股票代碼 | 是 | 四碼代號 |
| 動作 | 是 | BUY（買進）或 SELL（賣出） |
| 價格 | 是 | 實際成交價 |
| 張數 | 是 | 幾張 |
| 交易日期 | 是 | 實際成交日期 |
| 進出場理由 | **強烈建議** | 你為什麼買進 / 賣出（最重要的欄位） |
| 情緒標記 | 建議 | 這次決策時的心理狀態 |
| 損益 | 賣出時填 | 這筆交易實際賺或虧了多少元 |

### 買進交易會自動同步到持股監控

新增交易日誌時，如果動作是 `BUY`，系統會自動把這筆買進同步到「持股監控」：

- 如果該股票尚未在持股監控中，會新增一筆持股
- 如果已經持有同一檔股票，會用加權平均方式更新成本與張數
- 每次打開交易日誌頁，系統也會檢查過去尚未同步的未平倉買進紀錄，補回持股監控
- 手動在持股監控建立的資料不會被交易日誌同步覆蓋

這個設計是為了避免「交易日誌有買進，但持股監控忘了加入」的落差。若你用交易日誌記錄所有買賣，持股監控會自動跟上目前仍未平倉的部位。

### 情緒標記的選項與意義

| 情緒 | 描述 | 常見的錯誤 |
|------|------|-----------|
| 冷靜 | 按照計畫執行 | — |
| 樂觀 | 對後市有信心 | 可能低估風險 |
| 貪婪 | 想賺更多而加碼 | 放棄停損、擴大部位 |
| 恐慌 | 害怕繼續虧損 | 在低點砍在最低 |
| 衝動 | 沒有分析就進場 | 沒有計畫的買進 |
| 猶豫 | 一直在觀望 | 好機會出現卻沒進場 |
| FOMO | 怕錯過而追高 | 在過高的位置買進 |

長期記錄後，你會看到哪種情緒下的勝率最低，這就是你的心理弱點，需要特別注意。

### 三個核心績效指標

**指標一：勝率**

```
勝率 = 獲利交易次數 ÷ 總交易次數 × 100%
```

常見誤解：很多人以為要勝率 > 50% 才算好。實際上，只要盈虧比夠高，40% 的勝率也能穩定獲利。

**指標二：盈虧比（Profit Factor）**

```
盈虧比 = 所有獲利交易的總獲利 ÷ 所有虧損交易的總虧損
```

| 盈虧比 | 評估 |
|-------|------|
| < 1 | 虧損的總額大於獲利，長期必虧 |
| 1 - 1.5 | 僅達損益平衡，還需改善 |
| **1.5 - 2** | **良好，可以持續獲利** |
| > 2 | 優秀 |

**指標三：期望值**

```
期望值 = 勝率 × 平均獲利 - (1 - 勝率) × 平均虧損
```

期望值 > 0 = 這個策略長期下來會獲利。

以下用一個表格展示不同勝率和盈虧比組合的期望值：

| 勝率 | 盈虧比 | 每次期望值（假設虧損 10,000 元） |
|------|-------|------------------------------|
| 30% | 3:1 | +1,200 元 ✅ |
| 40% | 2:1 | +4,000 元 ✅ |
| 50% | 1.5:1 | +2,500 元 ✅ |
| 55% | 1:1 | +1,000 元 ✅ |
| 60% | 0.8:1 | -800 元 ❌ |

**重點：高勝率不代表獲利，盈虧比才是決定性因素。**

## 第十五章：五大交易紀律

> 這五條紀律是長期獲利的根基，不是選項，是必須遵守的規則。

---

### 紀律一：嚴格執行停損

**規則：** 進場前設好停損，到點必須執行，不能找任何理由拖延。

**常見的自我欺騙：**
- 「再等一下，它會回來的」→ 不一定，等的過程中虧損繼續擴大
- 「我已經虧這麼多了，現在賣不划算」→ 這是錯誤的沉沒成本思維，現在的損失是過去的決定造成的，未來的損失是你現在的決定決定的
- 「這公司基本面很好，一定會漲回來」→ 技術面和基本面是不同的時間框架，基本面回來可能需要 1-2 年，那段時間的資金成本是真實的損失

**停損是一種保費，不是失敗：** 每次停損，你是在為自己的帳戶繳一筆保費，保護它不受更大的損失。

---

### 紀律二：先看大盤再選股

**規則：** 開盤前必須先確認市場環境，空頭或偏空環境下不主動建立新倉位。

**為什麼：** 即使選股雷達找到了分數 90 分的股票，在空頭環境下進場的期望值仍然是負的。環境的影響力遠大於個股選擇。

---

### 紀律三：部位大小要計算，不要憑感覺

**規則：** 每次進場前，必須用「風險控制」頁面計算建議張數，不能因為「這次很有把握」就增加張數。

**最危險的話：** 「這次感覺特別確定，我要多買一些。」

有把握的感覺往往在你最接近虧損頂點的時候最強烈。部位大小的計算是為了保護你免受自己的過度自信影響。

---

### 紀律四：虧損時絕對不加碼（不攤平）

**規則：** 買進的股票跌了，絕對不能再買更多來「降低成本」。

**攤平的問題：**
- 你在一個正在虧損的標的上投入更多資金
- 如果繼續跌，虧損的絕對金額更大
- 最終止損時，損失可能是原本的 3-5 倍

**攤平和加碼的差別：**
- **攤平**：虧損後買更多 → 絕對禁止
- **加碼**：獲利且趨勢確認後買更多 → 可以考慮，但需要計算風險

---

### 紀律五：不懂的股票不買

**規則：** 對於買進的每一檔股票，你需要能夠清楚說出：「我買它的理由是____，我的停損在____，我的目標是____。」

如果這三個空格你無法填寫，這筆交易就是在賭博，不是在投資。

**不懂不代表要研究到所有財報，而是：**
- 知道為什麼技術面符合條件
- 知道這個產業最近的大方向
- 知道自己的風險承受範圍

## 第十六章：工具限制與風險提示

### 技術面的限制

**1. 資料非即時**

FinMind 免費帳號提供的是盤後資料，通常在收盤後 30-60 分鐘更新。這套工具**不適合當天買當天賣（當沖）**，適合次日進場的波段策略。

**2. 指標是落後的**

所有技術指標都基於過去的價格，訊號出現時，最好的進場點往往已經過了一段。實際進場需要自己判斷合理的進場時機，不是看到訊號就直接追高。

**3. 假突破的風險**

即使量能放大、均線向上，也可能發生「假突破」——短暫突破後立刻回落。這是技術分析無法完全避免的情況，停損是對抗假突破的唯一工具。

---

### 市場面的風險

**1. 黑天鵝事件**

技術分析無法預測突發事件：地緣政治衝突、重大財報地雷、政策急轉彎等。這些事件可能在毫無技術警示的情況下讓股票急跌。**停損是唯一的防護機制。**

**2. 主力操控風險**

部分台股個股存在主力（大戶、機構）刻意操控股價的情況。主力出貨時可能製造虛假的良好技術型態，吸引散戶進場後立刻拉高出貨。

識別高風險的跡象：
- 交易量異常放大但沒有新聞
- 股價在短期內異常快速上漲
- 股本較小、流通性較低的小型股

**3. API 使用限制**

- FinMind 免費帳號（註冊會員）：每小時 600 次上限
- 系統已內建本機快取機制，掃描時優先讀快取，正常情況幾乎不消耗 API 額度
- 若真的超過上限（HTTP 429），自動切換 yfinance 備援，並顯示橘色警告
- 如需快速填滿快取，至「6 - 資料管理」頁面啟動「全速重建模式」

---

### 心理風險（最被低估的風險）

**連續虧損的影響**

即使策略正確，連續虧損 3-5 筆是完全正常的事（55% 勝率的策略，每 10 筆會有 4-5 筆虧損）。連續虧損後：
- 很容易懷疑整個策略
- 容易在下一筆交易過度謹慎（錯過好機會）或過度激進（想快點回本）

**應對方式：** 看統計，不看單筆。評估策略的最小樣本是 20-30 筆，不要用 3-5 筆的結果做判斷。

## 第十九章：個股分析 — 逐項檢視單一股票條件

### 功能說明

在選股雷達找到候選標的後，進入「7 - 個股分析」可以針對**單一股票**深度確認 v3 / v4 技術條件是否真的達標，以及查看完整的圖表與數值細節。

> 個股分析是選股雷達的**確認工具**，不是選股工具。正確流程是：先用選股雷達篩出候選名單，再用個股分析逐項驗證。

### 查詢方式

在左側側邊欄輸入股票代號（例如 2330），選擇分析基準日後按「🔎 開始分析」。

| 設定項目 | 說明 |
|---------|------|
| 股票代號 | 6 碼以內，直接輸入代號 |
| 分析基準日 | 預設今日，可往前選擇查看歷史技術狀態 |
| 策略版本 | 可選 v4 領先攻擊版或 v3 均線突破版 |
| 主力連 3 日買超 | v3 / v4 皆會檢查此必要條件；來源是 Sponsor 分點主力快取 |
| 載入三大法人買賣超 | 選填，需消耗 API 額度，可查詢投信第一天買超與法人買超加分；不會載入分點主力 |
| 三線齊穿判斷模式 | v4 專用；嚴謹型（三線全部穿越）或寬鬆型（任一線即可） |

頁面頂部會顯示該股在分析基準日的**日成交量排行**，格式為「第 N / 全市場可排名檔數 名」。若自訂模式有設定「成交量前 N 名」條件，個股分析也會用這個排行判斷是否達標。

### 條件評分卡

頁面左側會依所選策略版本顯示逐項評分。v4 分為「必要條件」與「加分條件」兩組；v3 則用均線突破、量能、MACD/RSI、布林位置與主力連買來確認。

**v4 必要條件（七項全部達到才入選）**

| 條件 | 規則 | 分數 |
|------|------|------|
| 第一天站上 5/10/20MA | 今收 > 三均線；昨收在三線下方 | 35 分 |
| 均線糾結度 < 3%（昨日）| 昨日 (max − min) / MA20 < 3%，確認盤整後突破 | 20 分 |
| 量能爆發比 > 5 日均量 1.5 倍 | 今日成交量 > 5 日均量 × 1.5 | 15 分 |
| ATR 過熱防護通過 | 今收未超過 MA20 + 3.5 × ATR14 | 10 分 |
| 相對強度 RS > 80 | 過去 63 交易日 RS 分數 > 80，個股明顯領先大盤 | 10 分 |
| 突破 60 日收盤新高 | 今收 > 過去 60 個交易日最高收盤 | 10 分 |

| 不低於布林下軌 | 今日收盤 >= 布林下軌 |
| MACD 或 RSI 轉強 | MACD 黃金交叉/多頭，或 RSI 位於 50-70 健康區間 |
| 主力連 3 日買超 | 最近 3 個交易日 `broker_main_force_cache` 的主力買賣超皆為正 |

### 圖表區

頁面右側顯示日K線圖，包含：
- K 線（紅漲綠跌，台灣習慣色系）
- MA5 / MA10 / MA20 / MA60 均線
- 布林通道（上下軌 + 填色區域）
- 布林帶寬百分比
- 成交量（含顏色分紅綠）
- 主力買賣超長條圖（買超向上、賣超向下）
- RSI(14)（含 70/30 參考線）

**主力買賣超的定義：**

```
主力買賣超 = 前 15 大買超券商的總買超張數 - 前 15 大賣超券商的總賣超張數
```

數值為正代表主力買超，長條圖向上；數值為負代表主力賣超，長條圖向下。由於 FinMind 的券商分點資料一次只能查單一股票、單一日期，系統會把計算後的每日結果存入 `broker_main_force_cache`，後續查看同一股票同一日期時優先讀本機快取。

頁面下方另有**週K線圖**，含週MA10與扣抵值標示（紅色虛線，對應加分條件「週線扣抵值低位」的計算基準點）。

### 技術指標數值彙整

在圖表下方提供一張數值摘要表，列出所有計算中間值：

- MA5 / MA10 / MA20 / MA60
- 布林上軌 / 中軌 / 下軌 / 頻寬

- RSI(14)、MACD / Signal

### 歷史回溯模式

在左側選擇**過去的日期**，系統會：
1. 將所有技術條件與圖表切回該日期的收盤狀態
2. 在頁面底部額外顯示**後續走勢**，含 +1/+3/+5/+10 日報酬、前 10 日最高漲幅與最大回撤
3. 法人、融資與分點主力在歷史模式下優先讀取本機快取，避免混入今日 API 結果；若快取不足，條件會顯示未載入或資料不足。

> 這個功能適合用來「驗題」：看看當天技術條件達標的股票，後來實際走勢如何，幫助你建立對策略的直覺。

### Premium 摘要區（Sponsor 試用）

若啟用 FinMind Sponsor / Premium 功能，個股分析頁會在技術條件之外顯示 Premium 摘要。這個區塊以本機快取與頁面已載入資料為主，不會為了摘要額外批次呼叫 Premium API。

摘要會將訊號分成：

- 正向訊號：例如大戶比例改善、主力連買或集中度提高。
- 負向訊號：例如官方風險旗標、大戶比例下降、小股東比例上升、主力反手或賣超。
- 缺資料欄位：表示目前 cache 中沒有足夠資料，不能把它解讀成條件不符合。

股權分散資料若本期與前期比例持平，系統會略過不顯示；持平不是缺資料，也不是負向訊號。

### 注意事項

- 個股分析的日K與歷史模式讀取**本機快取**，若快取無資料會提示去「資料管理」頁面更新
- 個股分析會依 Sidebar 選擇顯示 v3 或 v4 評分卡
- 個股分析除了策略必要條件，也會參考目前儲存的自訂模式條件
- 三大法人、融資與主力買賣超需有快取或另行載入；手動載入會消耗 FinMind API 額度

## 第二十章：FinMind Sponsor Premium 試用功能

### 功能定位

Sponsor Premium 功能是這套工具的「增強層」，不是核心策略本身。v3 / v4 的技術條件、掃描流程、持股監控與風控原則都可以在 Free 模式下運作。Premium 資料只用來補強三件事：

- 風險辨識：官方處置、暫停交易、漲跌停等風險旗標。
- 籌碼觀察：分點主力、股權分散、大戶與小股東比例變化。
- 試用期評估：用歷史掃描結果驗證 Premium 訊號是否真的改善後續報酬。

### 降級原則

系統採用「可關閉、可降級、可觀察」的設計：

- Premium 未啟用或 tier 為 Free 時，不會硬打 Premium-only API。
- quota 過低或 API 回傳權限錯誤時，系統會進入 degraded 狀態，改讀本機 cache 或略過 Premium 欄位。
- 缺 Premium 資料不等於條件不符合，只代表目前不能用這個欄位判斷。
- 選股雷達的大量掃描不會逐檔呼叫 Premium API，避免快速耗盡額度。

### Known Issues / 後續確認

- `TaiwanStockShareholdingTransfer`（申報轉讓）目前列入管線與顯示，但實際資料來源仍需確認。若個別股票新聞有申報轉讓、FinMind 回傳沒有資料，先不要把它視為程式漏抓。
- `TaiwanStockPriceLimit` 已列入 Premium gate，但目前不再顯示成持股監控的 `[Premium] 官方風險旗標：price_limit`。後續若要使用，應改成清楚文字，例如「達漲停/跌停價」或「漲跌停參考價」。
- 股權分級 `_holding_level_bounds` 若遇到 FinMind 回傳純數字 level，會回傳 `(n, n)`。這是安全預設，該筆不會被錯誤歸到大戶或小股東區間，後續可視實際資料再細化。

### 分數欄位怎麼看

選股雷達現在保留原本的 `score` 相容性，同時拆出更清楚的欄位：

```text
final_score = base_score + premium_score - risk_penalty
```

- `base_score`：原本 v3 / v4 技術面分數。
- `premium_score`：預留給正向 Premium 訊號的加分欄位。
- `risk_penalty`：基本面或官方風險造成的扣分。
- `final_score`：實際顯示與排序用的分數。
- `premium_positive_flags`：正向 Premium 訊號文字。
- `premium_negative_flags`：負向訊號文字，可能包含基本面扣分，因此不是純 Premium。
- `premium_missing_fields`：目前缺資料的 Premium 欄位。

### 試用期評估怎麼用

進入「選股雷達 → 歷史紀錄」，上方會看到 Premium 試用期評估。建議用法：

1. 先累積至少數次掃描紀錄。
2. 選擇納入最近幾次掃描。
3. 比較 `negative risk/fundamental flags`、`positive premium flags`、`no tracked flags` 三組的 5 / 10 / 20 日後續報酬與勝率。
4. 觀察有效樣本數，不要用太少樣本做結論。

這個報表的目的不是直接產生買賣訊號，而是回答「Sponsor Premium 資料是否值得繼續使用」。

# 附錄：常見問題 Q&A

**Q：選股雷達選出來的股票可以直接買嗎？**

A：不行。選股雷達是「候選清單」，提供進一步研究的方向。在實際買進之前，你還需要：
1. 確認大盤環境是多頭或偏多
2. 看 K 線圖，確認進場點合理（不要追高）
3. 找到合理的停損位置
4. 用風險控制計算應該買幾張

**Q：分數高的股票，是不是就一定比分數低的股票好？**

A：分數高代表符合更多技術條件，但不代表一定會漲。分數是優先研究的順序，不是買進的保證。

**Q：系統發出危險警示，一定要馬上賣嗎？**

A：警示是提醒你「這個情況需要你做決定」，不是強制命令。但如果是「跌破停損價」的警示，這是你事先設好的底線，建議嚴格執行，不要找理由拖延。

**Q：選股雷達一個結果都沒有怎麼辦？**

A：這是好事，不是壞事。代表目前市場中沒有符合嚴格條件的股票，最好的操作就是「等待」。強迫進場才是危險的。

**Q：LINE 沒有收到推播怎麼辦？**

A：按順序確認：
1. 電腦是否開著？
2. 排程視窗（`python scheduler/jobs.py`）是否還在運行？
3. LINE Bot 是否還是好友？
4. 可到「市場環境」頁面的「排程控制」手動測試推播

**Q：可以同時持有幾檔股票？**

A：取決於帳戶大小和市場環境。一般建議：
- 多頭環境：3-5 檔
- 偏多震盪：2-3 檔
- 偏空或空頭：0-1 檔（或完全空倉）

每增加一檔持股，就要確保每檔的帳戶佔比不超過 25%，且總曝險不超過 80%。

**Q：工具說「偏多震盪」，但我看到一檔分數很高的股票，可以買嗎？**

A：可以評估，但要比多頭環境更謹慎：
- 部位縮小（降到 1-1.5% 帳戶風險）
- 停損設得比平常緊
- 只選分數最高的那 1-2 檔，不要同時追多檔

## 第十七章：資料管理 — 快取與背景抓取

### 為什麼需要本機快取？

FinMind 免費帳號每小時只有 600 次 API 請求；Sponsor 帳號可用額度較高，目前系統會依 `user_info` 回傳額度調整，Sponsor 盤後/非交易時間可用到每小時 6000 次。全市場約 950 檔普通股，每次掃描若都從網路抓取，很快就會超出限制。

系統的解法是：**把歷史股價存在你的電腦裡**（本機快取），掃描時先讀本機資料，只有遇到新資料才呼叫 API。

### 背景預抓取工作器

App 啟動後，系統會自動在背景開始下載全市場的歷史價格資料。你不需要做任何事，它會靜靜地跑在背景。

| 時段 | 每小時上限 | 說明 |
|------|----------|------|
| 交易時間（09:00–盤後切換前） | 100 次 | 保留額度給手動操作 |
| 盤後 / 非交易時間 | Free 600 次；Sponsor 依帳號額度，最高 6000 次 | 全力填充快取 |
| 全速重建模式 | 依帳號額度 | 開放全部額度，手動啟動 |

系統不再固定用單一的盤後切換時間。它會記錄「當天盤後第一筆成功更新時間」，之後自動把全速開始時間往前提早 10 分鐘，逐步貼近資料源實際更新節奏。

### 休市模式

連假或停市期間，法人資料不會更新。開啟「🏖️ 休市模式」可讓法人快取永不過期，避免反覆呼叫 API 浪費配額。

- **開啟位置：** 「6 - 資料管理」頁面最上方的切換按鈕
- **效果：** 法人資料快取無限期有效，不消耗 API 額度
- **恢復交易後：** 手動關閉此開關，下次掃描時系統會自動更新法人資料

### 交易日判斷與參考日期

資料管理頁現在會先判斷「今天是否應該有交易資料」，再決定狀態文字與進度基準：

- 週六、週日會顯示非交易日，不會誤判成交易時間
- 開啟休市模式時，會視為手動休市，不會要求今日法人或融資資料
- 一般交易日會用 `resolve_latest_trading_day()` 推估最新應有資料日
- 下方「資料待更新」與進度條會以最新交易日為基準，不再只看工作器記憶體中的暫時計數

這樣可以避免週末看到「等待 FinMind 更新今日資料」或「交易時間中」這類誤導訊息。

### 快取品質診斷怎麼看

「🔍 快取品質診斷」會把本機資料分成 5 類，幫你快速找出為什麼某些股票掃不到、回測跑不動、或資料明顯不完整：

| 分類 | 意義 | 你應該怎麼做 |
|------|------|-------------|
| 完全缺失 | 股票清單裡有這檔，但本機完全沒有任何價格快取 | 用「手動補抓」或啟動重建模式 |

### 市場指數資料（TAIEX）為什麼重要

回測模組現在會使用 **TAIEX（台灣加權股價指數）** 當作市場基準。這筆資料主要用在：

- 比較策略績效是否優於大盤
- 支援相對強弱（RS）或市場基準相關的計算
- 避免基準曲線空白，或因市場資料缺失導致報告不完整

如果資料管理頁看到「📈 市場指數下載（TAIEX）」尚未完成，建議先補齊，再執行正式回測。否則即使個股資料完整，部分報表或比較基準仍可能缺值。

### 每日自動更新流程

每個交易日，背景工作器會自動依序完成以下階段：

| 時間 | 階段 | 說明 |
|------|------|------|
| 全天 | **① OHLCV 核心價格** | 持續補齊近 400 天 K 線資料，交易時間限速 100 次/小時，盤後依帳號額度全速 |
| 13:45 | **② Yahoo Bridge** | 收盤後 15 分鐘以 Yahoo Finance 補充今日收盤價（約 15 分鐘延遲），供盤後早期使用 |

### 為什麼法人/融資進度晚於 OHLCV？

法人籌碼與融資融券是「Supplementary 補充階段」，有兩個前提條件才會啟動：

1. **OHLCV 全部完成**：待更新清單（`needs_update`）清空後，才進入補充階段
2. **FinMind 已發布今日資料**：`resolve_latest_trading_day()` 確認 FinMind 的最新資料日期 ≥ 今日

因此，若 OHLCV 還有待更新股票，法人/融資進度條會顯示「⏳ 等待 OHLCV 核心資料完成（還剩 X 檔）」。這是正常現象，不代表系統異常。

若法人或融資開始補抓後出現個別股票失敗，資料管理頁會列出失敗代號。常見原因包含資料源尚未收錄、該股票當日沒有對應資料、或 API 暫時回傳空資料。這些失敗會和「FinMind 回傳無資料、可略過」分開顯示，方便你判斷是否需要手動補抓。

### 失敗股票與自動略過

若某檔股票長期抓不到新資料，系統會自動區分：

- **完全無資料**：例如特別股、權證或資料源未收錄，會從背景重試清單中排除

這個機制的目的是把 API 配額留給真正還有更新價值的股票。

## 第十八章：回測模組 — 驗證策略歷史績效

### 功能說明

回測模組讓你用歷史資料**模擬**選股策略的過去表現，驗證策略邏輯是否有效，避免用未經驗證的條件進行實際交易。

> **重要提醒：** 歷史績效不代表未來結果。回測是用來「剔除明顯無效的策略」，而非「保證未來獲利」。

### 開始前：確認資料就緒

回測模組**不提供內建下載按鈕**，所有歷史資料需透過 **6 - 資料管理** 頁面補充：

- 前往「6 - 資料管理」→ 啟動「📼 回測歷史資料重建模式」，系統會在背景自動下載最多 10 年的全市場日K資料
- 回測模組頁面頂端會即時顯示**加權指數（TAIEX）快取狀態**，確認大盤資料正常後再執行

### 兩個步驟

**步驟一：設定參數並執行**（⚙️ 設定回測參數 頁籤）

| 參數 | 說明 | 預設值 |
|------|------|--------|
| 回測期間 | 起始與結束日期 | 最近一年 |
| 使用股票數 | 從本機快取取前 N 檔 | 100 |
| 最低強度分數 | 只買分數達到門檻的訊號 | 65 分 |
| 三線齊穿模式 | 嚴謹型或寬鬆型進場判斷 | 嚴謹型 |
| 個股最大 BIAS | 避免追過熱個股 | 10% |
| 大盤 MA20 濾網 | 大盤條件不佳時暫停進場 | 開啟 |

### 資金管理參數

最近版本的回測不再只是「每次看到訊號就固定買 1 檔」。系統改成更接近真實交易的資金管理模型，會根據帳戶大小、剩餘現金與單筆風險限制來決定是否進場與買多少股。

| 參數 | 說明 | 預設值 |
|------|------|--------|
| 初始資金 | 回測起始帳戶金額，所有績效以此為基準 | 1,000,000 元 |
| 最大持倉檔數 | 同一時間最多持有幾檔股票 | 5 檔 |
| 單筆風險 (%) | 每筆交易最多願意承擔帳戶淨值的幾 % | 2% |
| 買進手續費 | 買進成交成本 | 0.1425% |
| 賣出手續費 | 賣出成交成本 | 0.1425% |
| 證交稅 | 台股賣出交易稅 | 0.3% |

### 系統如何決定買幾股

回測引擎會先估算這筆交易的停損距離，再用「單筆風險預算」反推出可買股數。簡化理解如下：

1. 先算出這筆交易每股最多可能虧多少
2. 用帳戶淨值 × 單筆風險 %，得到這筆交易可承受的最大虧損金額
3. 再同時檢查剩餘現金是否足夠
4. 最後向下取整到台股整張單位（1000 股）

這代表同樣是 100 萬資金，停損比較遠的股票，系統會自動買比較少；停損比較近、風險較小的股票，才有空間買比較多。若同一天有多檔候選標的，系統也會依訊號強度排序，優先把資金分配給分數較高者。

**步驟二：查看績效報告**（📊 查看回測結果 頁籤）
- 勝率、盈虧比、最大回撤、累積報酬等核心指標

### 報告新增了哪些欄位

新版績效報告改成以「帳戶淨值」而不是單純百分比累乘來計算，因此你會看到更多接近實際資金管理的欄位：

| 欄位 | 說明 |
|------|------|
| 初始資金 | 回測一開始投入的本金 |
| 最終淨值 | 回測結束時，現金加上持倉市值的總和 |
| 總損益 | 最終淨值減掉初始資金的絕對金額 |
| 最大回撤（MDD） | 從淨值高點回落的最大幅度 |
| 股數 | 每筆交易實際買進的股數 |
| 成本 / 賣出收入 | 已把手續費與證交稅計入後的真實成本與回收金額 |
| 進場分數 | 該筆交易進場當天的候選強度分數，方便回頭檢查排序邏輯 |

如果你看到某檔股票明明訊號成立卻沒有成交，通常不是 bug，而是以下幾種真實限制之一：

- 當下可用現金不足
- 已達最大持倉檔數
- 按照風險預算換算後，連 1 張都買不起
- 同日有更高分的候選股，資金先被分配掉

### 出場規則：移動式停損/停利

回測引擎使用**移動式**出場規則，比傳統固定停損更能讓利潤奔跑：

#### 移動停損（預設 8%）

傳統固定停損從「買入價」計算，一旦設定就不動。

移動停損改為從**持倉期間最高價**計算：
```
停損觸發價 = 持倉最高價 × (1 - 停損幅度%)

例：買入 100 元，股票漲到 120 元
  → 停損線從「92 元」移到「110.4 元」（120 × 0.92）
  → 後來跌回 110.4 → 自動出場，保住 10.4 元的利潤
```

**優點：** 股票繼續漲時，停損線跟著上移，保護已累積的帳面獲利。

#### 移動停利（預設啟動門檻 15%，啟動後追蹤 5%）

當股票達到啟動門檻後，追蹤幅度自動縮緊，更積極鎖住利潤：

```
未達 +15%：使用移動停損（8% 追蹤）
達到 +15% 後：切換為移動停利（5% 追蹤）

例：買入 100 元，漲到 120 元（+20%，已超過 15% 啟動門檻）
  → 移動停利線 = 120 × 0.95 = 114 元
  → 後來跌回 114 → 出場，鎖住 14 元的利潤
```

#### 其他出場條件

| 條件 | 說明 |
|------|------|
| 跌破 MA5 / MA10 / MA20 | 可自行選擇哪條均線作為跌破出場條件（可關閉） |
| 持滿 N 天 | 最長持有天數限制，到期強制出場（預設 20 天） |

#### 出場原因分析

| 出場原因 | 意義 |
|---------|------|
| 🔥 移動停損 | 從最高點下跌超過停損幅度出場（保護性出場） |
| ✅ 移動停利 | 達到啟動門檻後緊縮追蹤，鎖住利潤出場 |
| 跌破MA5 / MA10 / MA20 | 依你在回測頁設定的均線週期提前離場 |
| 持滿N天 | 超時強制出場 |

### 回測的常見誤解

**誤解一：「回測績效 80%，未來也會有 80%」**
回測是用歷史資料優化的，存在「過擬合」風險。建議保守解讀，用來排除明顯差的策略，而非預測確切的未來績效。

**誤解二：「回測沒有交易成本，不準確」**
本系統的回測**已內建台股交易成本**：
- 買進手續費：0.1425%
- 賣出手續費：0.1425%
- 交易稅（賣出）：0.3%
- 合計約 0.447%，已接近實際環境

**誤解三：「用越多年的資料越準確」**
資料越舊，市場結構可能已改變（例如 2018 年以前的台股結構和現在差異很大）。建議重點看最近 2-3 年的表現，參考 5-10 年的趨勢。

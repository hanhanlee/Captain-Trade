## 第十三章：LINE 自動推播設定

### 推播種類與時間

| 時間 | 推播內容 | 觸發條件 |
|------|---------|---------|
| 週一至週五 13:30 | 持股盤中警示 | 有任何警示訊號才發送 |
| 週一至週五 14:35 | 持股盤後警示 | 有任何警示訊號才發送 |
| 週一至週五 14:45 | 選股雷達結果前 5 名 | 每日固定發送 |
| 週五 15:10 | 週績效摘要 | 每週固定發送 |

### 推播格式範例

**選股雷達推播：**

```
📊 今日選股雷達 — 強勢候選股

1. 2330 台積電
   收盤：850 元  🔴+2.3%
   量比：1.8x  分數：85
   [站上MA20、MA20向上、量增、MACD黃金交叉、布林正常]

2. 2317 鴻海
   ...
```

**持股警示推播：**

```
💼 持股監控警示

🔴 2330 台積電
   跌破停損價 780 元
   現價 775 元  損益 -8.2%

🟡 2454 聯發科
   從高點回撤 9.5%
   現價 820 元  損益 +3.1%
```

### 手動觸發推播

如果不想等自動時間，可以在「市場環境」頁面的「排程控制」頁籤，隨時手動觸發任何推播任務。

### 訂閱者管理（新版）

現在推播不再只綁定單一 `LINE_USER_ID`。系統改成：

- **群播模式**：所有 `已啟用` 的訂閱者都會收到選股、警示與週報
- **單人測試**：可在「6 - 資料管理」頁面對單一訂閱者送測試訊息
- **停用不中斷紀錄**：暫停接收時，不需要刪除資料
- **向下相容**：若資料庫沒有任何訂閱者，仍會 fallback 到 `.env` 的 `LINE_USER_ID`

你可以在「6 - 資料管理」頁面：

- 查看所有訂閱者
- 個別啟用 / 停用群播
- 新增或刪除 LINE User ID
- 立即送出測試訊息確認 Bot 是否正常

### Webhook 自動訂閱

如果你有啟動 `webhook.py` 並在 LINE Developers Console 設定 Webhook URL，使用者可以透過 Bot 自動加入或暫停群播：

| 事件 / 指令 | 系統行為 |
|------------|---------|
| 加 Bot 好友 | 自動加入訂閱者清單 |
| 傳送任意訊息 | 若尚未訂閱，會自動加入 |
| `我的ID` | 回覆自己的 LINE User ID |
| `取消訂閱` | 停用群播，但保留紀錄 |
| `重新訂閱` | 重新啟用群播 |

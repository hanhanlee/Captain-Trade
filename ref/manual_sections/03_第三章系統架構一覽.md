## 第三章：系統架構一覽

### 五個模組的關係

```
市場環境（整體環境判斷）
        ↓
選股雷達（找候選標的）→ 個股分析（逐項確認技術條件）
        ↓
風險控制（計算進場張數）
        ↓
[進場買股]
        ↓
持股監控（持倉期間監控）← LINE 推播
        ↓
交易日誌（記錄 → 統計 → 學習）
```

正確的操作流程應該從上到下，每個環節都不能跳過。個股分析是選股雷達的補充工具，找到候選標的後可以進入個股分析頁逐項確認技術細節。

### 技術架構（給想了解細節的人）

| 元件 | 技術 | 說明 |
|------|------|------|
| 操作介面 | Streamlit | 網頁型儀表板，瀏覽器開啟即可使用 |
| 股市資料來源 | FinMind API | 台股公開資料（日K、法人買賣、籌碼），600次/小時 |
| 備援資料源 | yfinance | FinMind 達限額時自動切換，法人條件暫停 |
| 技術指標計算 | 純 Python / pandas | MA、MACD、RSI、布林通道，自行實作無套件依賴 |
| 資料儲存 | SQLite | 本機資料庫，儲存持股、交易日誌、歷史價格快取 |
| 背景預抓取 | 自製排程執行緒 | App 啟動後自動在背景填充全市場價格快取 |
| 推播通知 | LINE Messaging API | 盤後自動推播到你的 LINE |
| 排程系統 | APScheduler | 設定固定時間自動執行掃描與推播 |

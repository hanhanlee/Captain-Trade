## 第十九章：個股分析 — 逐項檢視單一股票條件

### 功能說明

在選股雷達找到候選標的後，進入「7 - 個股分析」可以針對**單一股票**深度確認 v4 技術條件是否真的達標，以及查看完整的圖表與數值細節。

> 個股分析是選股雷達的**確認工具**，不是選股工具。正確流程是：先用選股雷達篩出候選名單，再用個股分析逐項驗證。

### 查詢方式

在左側側邊欄輸入股票代號（例如 2330），選擇分析基準日後按「🔎 開始分析」。

| 設定項目 | 說明 |
|---------|------|
| 股票代號 | 6 碼以內，直接輸入代號 |
| 分析基準日 | 預設今日，可往前選擇查看歷史技術狀態 |
| 載入法人買賣超 | 選填，需消耗 API 額度，可查詢投信第一天買超條件 |
| 三線齊穿判斷模式 | 嚴謹型（三線全部穿越）或寬鬆型（任一線即可） |

### v4 條件評分卡

頁面左側顯示逐項評分，分為「必要條件」與「加分條件」兩組：

**必要條件（四項全部達到才入選）**

| 條件 | 規則 | 分數 |
|------|------|------|
| 第一天站上 5/10/20MA | 今收 > 三均線；昨收在三線下方 | 35 分 |
| 均線糾結度 < 3% | (max − min) / MA20 < 3% | 20 分 |
| 量能爆發比 > 前日 1.5 倍 | 今日成交量 > 昨日 × 1.5 | 15 分 |
| 股價乖離 MA20 < 5% | (收盤 − MA20) / MA20 < 5% | 10 分 |

**加分條件（最多 +50 分）**

| 條件 | 分數 |
|------|------|
| 布林頻寬縮減（vs 20 日前） | +10 |
| 投信第一天買超 | +10 |
| 突破近 20 日收盤高點 | +8 |
| 週線 MA10 扣抵值低位 | +10 |
| 融資減少 / 籌碼集中 | +5 |
| 相對強度 RS > 70 | +7 |

每個條件都顯示 ✅ / ❌ 與實際數值，讓你知道差了多少才達標，而不只是「過/沒過」。

### 圖表區

頁面右側顯示日K線圖，包含：
- K 線（紅漲綠跌，台灣習慣色系）
- MA5 / MA10 / MA20 / MA60 均線
- 布林通道（上下軌 + 填色區域）
- 成交量（含顏色分紅綠）
- RSI(14)（含 70/30 參考線）

頁面下方另有**週K線圖**，含週MA10與扣抵值標示（紅色虛線，對應加分條件「週線扣抵值低位」的計算基準點）。

### 技術指標數值彙整

在圖表下方提供一張數值摘要表，列出所有計算中間值：

- MA5 / MA10 / MA20 / MA60
- 布林上軌 / 中軌 / 下軌 / 頻寬
- RSI(14)、MACD / Signal
- 均線糾結度（%）、BB頻寬變化方向

### 歷史回溯模式

在左側選擇**過去的日期**，系統會：
1. 將所有技術條件與圖表切回該日期的收盤狀態
2. 在頁面底部額外顯示**後續走勢**，含 +1/+3/+5/+10 日報酬、前 10 日最高漲幅與最大回撤
3. 法人與融資資料在歷史模式下自動略過（避免混入今日 API 資料）

> 這個功能適合用來「驗題」：看看當天技術條件達標的股票，後來實際走勢如何，幫助你建立對策略的直覺。

### 注意事項

- 個股分析讀取**本機快取**，若快取無資料會提示去「資料管理」頁面更新
- 個股分析固定使用 **v4 領先攻擊版**條件評分，不提供 v3 切換
- 法人買賣超需另行勾選才會載入，會消耗 FinMind API 額度

## 第八章：選股雷達 — 找值得買的股票

### 功能說明

每天掃描台股市場（可選 20 / 100 / 全市場檔），依技術分析條件篩選出符合標準的股票，依強度分數排名輸出。

### 掃描範圍選擇建議

| 選項 | 股票數量 | 所需時間 | 建議使用時機 |
|------|---------|---------|------------|
| 快速測試 | 20 檔 | 約 1 分鐘 | 初次使用確認功能正常 |
| 小型掃描 | 100 檔 | 約 5 分鐘 | **日常使用（推薦）** |
| 全市場掃描 | 約 950 檔 | 約 30-50 分鐘 | 每週一次深度掃描 |

> **注意：** FinMind 免費帳號（註冊會員）每小時限制 600 次 API 請求。系統背景會自動預先下載資料存入本機，掃描時優先讀本機快取，大幅減少 API 消耗。若額度不足，自動切換 yfinance 備援（法人條件暫停）。

> **Sponsor Premium 試用模式：** 若在 `config.toml` 啟用 FinMind Sponsor / Premium 功能，選股雷達仍以既有 v3 / v4 技術條件為核心。Premium 資料只用於補充風險旗標、基本面扣分、籌碼觀察與試用期評估；當 Premium 關閉、額度不足或 API 不可用時，系統會降級成讀取本機快取或略過 Premium 欄位，不會讓 Free 模式失效。

### 掃描模式與歷史回放

新版 Sidebar 多了四個實用功能：

1. **掃描模式預設**
- `⚡ 極速`：20 檔、最少額外條件，適合快速確認
- `🎯 標準`：100 檔、開啟法人條件，適合日常使用
- `🔬 完整`：全市場、可搭配融資與族群偵測，適合收盤後完整檢查
- `⚙️ 自訂`：把你常用的一整套設定存下來，下次一鍵套用

2. **分析基準日**
- 可選今天，或回放過去某一天
- 選過去日期時，系統會自動改成**歷史回溯模式**
- 歷史回溯模式只使用本機價格資料，不混入今天才知道的法人、融資或基本面資料

3. **三線齊穿模式**
- `嚴謹型`：昨日收盤必須在 MA5 / MA10 / MA20 三條線下方
- `寬鬆型`：昨日只要仍在三線中的任一線下方即可
- 嚴謹型標的較少但更乾淨；寬鬆型標的較多但雜訊也較高

4. **後續盤勢**
- 歷史回溯掃描完成後，頁面會直接列出 `+1 / +3 / +5 / +10` 日報酬
- 也會計算各期間的最高漲幅，方便快速檢查當天訊號後續表現

### v4 規則總覽

目前選股雷達使用的是 **v4 領先攻擊版**。核心思想不再是「已經走很久的強勢股」，而是找出：

> **均線剛糾結完成，今天才剛啟動突破的股票。**

#### ✅ 必要條件：七項全過才入選

| 條件 | 分數 | 實際判斷 |
|------|------|----------|
| 第一天站上 5/10/20MA | 35 | 今日收盤高於 MA5 / MA10 / MA20，且昨日仍在三線下方（依嚴謹/寬鬆模式判定） |
| 均線糾結度 < 3%（昨日）| 20 | `(max(MA5,MA10,MA20) - min(...)) / MA20`，**以昨日均線計算**，確認突破前處於盤整 |
| 量能爆發比 > 1.5 倍 | 15 | 今日成交量 > 5 日均量 × 1.5 |
| ATR 過熱防護通過 | 10 | 今日收盤未超過 `MA20 + 3.5 × ATR14`（倍數可在 Sidebar 調整） |
| 相對強度 RS > 80 | 10 | 過去 63 個交易日報酬率明顯跑贏大盤，RS 分數 > 80 |
| 突破 60 日收盤新高 | 10 | 今日收盤 > 過去 60 個交易日的最高收盤，確認為真實突破 |
| 主力連續買超 3 日以上 | 10 | 讀取 `broker_main_force_cache`，最近 3 個交易日「前 15 大買超分點 - 前 15 大賣超分點」皆為正 |

七項全部成立，基礎分就是 **110 分**。只要其中一項沒過，就不會進入候選名單。

> **均線糾結度為何改查昨日？** 突破當天的漲幅往往會直接撐開均線距離，若用今日數值反而會把最強的爆量突破股篩掉。改用昨日狀態，才能正確確認「盤整後才突破」。

> **為什麼主力連 3 日買超是必要條件？** v4 找的是「剛啟動且有資金確認」的股票。若只有價格突破、沒有主力資金連續承接，訊號容易變成假突破，因此目前主力連續買超 3 日以上已列為必要條件。這裡的「主力」不是三大法人合計，而是 Sponsor 分點券商資料計算出的主力買賣超；若 `broker_main_force_cache` 尚未補齊，這項會判定未達。

#### ➕ 加分條件：用來排序強弱

| 條件 | 分數 | 意義 |
|------|------|------|
| 布林頻寬縮減 | 10 | 今天的布林頻寬低於 20 天前，代表波動收斂後準備擴張 |
| 投信第一天買超 | 10 | 昨日投信不為正，今日轉正，表示新資金剛進場 |
| 週線 MA10 扣抵值低位 | 10 | 10 週前收盤低於當前週 MA10，代表週線均線未來有自然上揚條件 |
| 融資減少 / 籌碼集中 | 5 | 融資餘額較前一日下降，代表散戶去槓桿 |

**滿分 = 必要條件 110 + 加分 35 = 145 分。**

### v3 / v4 策略差異

選股雷達可以在 Sidebar 切換 **v4 領先攻擊版**與 **v3 均線突破版**。兩者不是同一套策略：

| 策略 | 核心用途 | 必要條件 |
|------|----------|----------|
| v4 領先攻擊版 | 找「盤整剛結束、今日啟動突破」的股票 | 三線齊穿、均線糾結、量能爆發、ATR 過熱防護、RS > 80、60 日新高、主力連 3 日買超 |
| v3 均線突破版 | 找「已站上趨勢、技術條件轉強」的股票 | 站上 MA20、MA20 向上、量增、不低於布林下軌、MACD 或 RSI 轉強、主力連 3 日買超 |

v4 比 v3 嚴格很多，因此常見狀況是：

- v3 有候選股，但 v4 沒有候選股
- v4 候選股數量很少，但條件更集中在「剛突破」與「資金確認」
- 兩者都會要求主力連續買超 3 日以上，因此 Sponsor 分點主力快取尚未更新時，兩個策略都可能沒有結果

結果表的「觸發訊號」會依策略版本分開顯示：選 v3 時只列 v3 相關訊號；選 v4 時只列 v4 相關訊號。這避免 V3 結果混入 V4 標籤，造成兩個策略看起來一樣。

### 補充條件與開關

這些條件不是 v4 核心必要條件，但會影響排序或前置過濾：

| 功能 | 用途 |
|------|------|
| 法人條件 | 主力連 3 日買超是必要條件，來源是 Sponsor 分點主力快取；三大法人設定只作為額外加分或結果確認 |
| 融資餘額 | 需額外 API，會把融資下降視為加分 |
| 基本面過濾 | 可用 EPS、現金流、ROE、負債比先排除品質不佳標的 |
| 流動性過濾 | 先從成交量前 N 名或日均量門檻縮小股票宇宙 |
| 產業輪動過濾 | 只掃近一週漲幅前 N 強的產業 |
| 族群偵測 | 可看 HP Density 與 Turnover Ratio，確認資金是否集中流入族群 |

> **歷史回溯模式提醒：** 歷史模式只回放價格資料，因此法人、融資與基本面條件會自動停用，避免把現在才知道的資料套進過去。

### 過熱股防護機制（ATR 動態門檻）

選股雷達雖然找啟動股，但仍會防止追得太高。新版改用 **ATR 動態門檻**，取代舊版固定的月線乖離率（%）：

```
過熱條件：今日收盤 > MA20 + N × ATR14
```

ATR（真實波動幅度）會自動隨個股波動特性縮放——波動大的半導體股門檻自然更寬；波動小的傳產股門檻自然更嚴，不會用同一把尺誤殺熱門領漲股。

Sidebar 可以設定：

1. **過熱 ATR 倍數**（預設 3.5）
   - 一般多頭環境維持 3.5
   - 半導體或熱門電子股噴發期可調至 4.5 ～ 5.0
   - 0：停用 ATR 過熱防護

2. **超過門檻時的處置**
- **直接剔除**：保守型做法
- **扣分懲罰**：保留觀察，但讓排名往後

---

### 分數怎麼看

| 強度分數 | 解讀 |
|---------|------|
| 110 分 | 剛好通過七項必要條件，屬於「成立但尚未特別強」 |
| 115-125 分 | 必要條件外，有一到兩項加分確認 |
| 130 分以上 | 必要條件完整，且多項加分同步成立，屬精選強勢候選 |

分數不是保證報酬，而是幫你把「形態最乾淨、資金確認較多」的股票排到前面。

### 產業族群分析（第二個頁籤）

除了個股排行榜，選股雷達還提供「產業族群分析」功能，幫你看懂資金流向。

**什麼是產業族群分析？**

當選股雷達同時選出 3 檔散熱模組股票、2 檔 AI 伺服器股票，這代表資金正在集中流入這兩個產業。這種現象叫做「族群輪動」——機構法人通常以「產業佈局」的方式進場，而不是隨機選股。

**如何解讀？**

| 顯示資訊 | 意義 |
|---------|------|
| 各產業入選檔數 | 入選越多，代表這個產業正在吸引更多資金 |
| 產業平均分數 | 分數越高，代表這個產業的整體技術面越強 |
| 各產業最強股 | 在最熱門的產業中，分數最高的就是最值得研究的標的 |

**操作建議：**

1. 找出入選最多的 1-2 個產業
2. 在那個產業中，選分數最高的 1-2 檔股票深入研究
3. 「最強產業裡的最強股票」是機構法人的基本操作邏輯

---

### 流動性前置過濾（掃描前先縮小範圍）

在開始掃描前，可以先用流動性條件過濾股票宇宙，聚焦真正值得關注的標的：

| 模式 | 說明 | 建議使用時機 |
|------|------|------------|
| **前日量前 N 名**（預設 100）| 鎖定昨天成交最活躍的 100 檔股票 | 日常使用，動態追蹤市場熱點 |
| **日均量 ≥ N 張**（預設 1000，上限 50,000）| 近 20 日平均量超過 N 張才考慮 | 想篩掉低流動性股票時使用 |
| 不過濾 | 掃描全部股票 | 週末深度掃描 |

> **為什麼用量能代替資本額？** 日均量超過 1000 張的股票，資本額幾乎必然超過 20 億，兩個條件高度重疊。而量能是每日動態更新的，比靜態的資本額更能反映當下市場關注度。

---

### 產業輪動過濾（只掃最強族群）

在流動性過濾之上，還可以再加一層「產業輪動過濾」：

**原理：** 計算所有已下載股票的近 5 日（一週）漲幅，按產業分組取平均，選出漲幅排名前 N 個產業，只在這些產業內做技術篩選。

**操作：** Sidebar 勾選「只掃描近一週漲幅前 N 類股」，預設取前 3 個產業。

**掃描完成後頁面會顯示：**
```
📊 本次鎖定產業（近 5 日漲幅前 3 名）：
  半導體 (+4.2%)　散熱模組 (+3.8%)　AI伺服器 (+3.1%)
```

**三層過濾疊加效果：**
```
全市場 950 檔
  → 前日量前 100 名（流動性過濾）
  → 近一週漲幅前 3 個產業（產業輪動過濾）
  → 技術分析篩選（必要條件全過）
  → 最終候選：5-15 檔精選個股
```

這種由上而下（Top-Down）的篩選邏輯，和機構法人的選股流程一致：**先選大環境 → 再選產業 → 最後選個股**。

### 掃描歷史紀錄（第四個頁籤）

每次掃描完成後，系統會自動將結果儲存到本機資料庫。在「歷史紀錄」頁籤中可以：

| 功能 | 說明 |
|------|------|
| 查看過去記錄 | 最新的在最上方，顯示掃描時間與找到的股票數量 |
| 展開查看條件 | 可看到當時使用的所有篩選條件（股價、量能、產業、RS 門檻等） |
| 結果預覽 | 展開後直接顯示前 10 筆股票 |
| 載入結果 | 把歷史結果還原到「掃描結果」頁籤，方便重新查看或比較 |
| 刪除紀錄 | 可刪除不再需要的歷史 |

**用途：** 可以回頭比較不同日期的掃描結果，觀察股票從候選到真正啟動的時間差，慢慢建立自己對「訊號出現到進場時機」的感覺。

**Premium 試用期評估：** 歷史紀錄頁籤上方會顯示「Premium 試用期評估」區塊。這個報表只讀取本機 scan history 與 price cache，不會呼叫 Premium API。它會把過去掃描結果分成：

- `negative risk/fundamental flags`：有官方風險、基本面扣分或其他負向訊號
- `positive premium flags`：有正向 Premium 籌碼或加分訊號
- `no tracked flags`：目前沒有被記錄的 Premium / 風險訊號

報表會比較各組在 5、10、20 個交易日後的平均報酬、勝率與有效樣本數，讓你在 Sponsor 試用期判斷 Premium 訊號是否真的改善選股結果。負向分組可能包含基本面扣分，不代表全部都是純 Premium 資料。

### 篩選漏斗（🔬 第二個頁籤）

掃描完成後，「篩選漏斗」頁籤會顯示每個條件在篩選過程中排除了多少股票，讓你清楚看到條件的嚴格程度。

**主要顯示內容：**

| 區塊 | 說明 |
|------|------|
| 漏斗長條圖 | 從「載入資料」到「通過所有條件」的逐層存活數量 |
| 各條件個別命中率 | 每個技術條件單獨觸發的比例（不是累積排除） |
| 近失標的（Near Miss）| 差一兩個條件就入選的股票，依缺失條件分組展示 |
| 前置過濾排除明細 | 因股價過低、日均量不足、資料不足而被排除的股票統計 |

**如何利用漏斗優化條件：**
- 若漏斗中某個條件排除了 90% 以上的股票 → 可能設得太嚴，考慮放寬
- 若某個加分條件命中率只有 5% → 對排名影響有限，可評估是否保留
- 「近失標的」是很好的觀察清單——這些股票幾乎符合，可手動複查是否值得追蹤

### 基本面過濾（財報品質篩選）

在 Sidebar 的「基本面過濾」區塊，可以選擇啟用財報條件，在技術篩選之前先排除虧損股與財務地雷。

| 條件 | 說明 |
|------|------|
| EPS TTM > 0 | 近四季合計每股盈餘為正（排除虧損公司） |
| 營業現金流 > 0 | 近四季合計營業現金流為正（排除假獲利） |
| 最低 ROE | 近四季股東權益報酬率下限（0 = 不限，建議 10-15%） |
| 負債比上限 | 最新季負債比例上限（0 = 不限，一般產業建議 < 60%） |

**快取機制：** 財報資料快取 90 天，首次啟用時需呼叫 API，之後讀本機快取，幾乎零額度消耗。

> **注意（免費帳號限制）：** FinMind 財報端點（`TaiwanStockFinancialStatements`）需要**付費方案**。免費帳號啟用基本面過濾時，系統會顯示橘色提示，自動跳過基本面篩選，選股結果不受影響。

### 族群集體突破偵測（🔥 新功能）

在 Sidebar「族群集體突破偵測」區塊可啟用兩個進階的族群動量指標，掃描後在「產業族群」頁籤的最底部顯示結果。

#### 指標一：族群創高密度（HP Density）

**原理：** 計算族群內有多少比例的股票今日收盤等於近 N 日最高收盤。

```
族群 A 有 20 檔股票
其中 8 檔今天收盤創了 20 日新高
→ HP Density = 8 ÷ 20 = 40%
```

**意義：** 一個族群裡有 40% 以上的股票同時創高，代表這個族群是「集體突破」，而非個別股票的偶然創高。這種族群性的動作通常代表機構法人正在整族群佈局。

**可調整參數：**
- 創高天數 N（預設 20 日）
- 族群創高比例門檻（預設 30%）

#### 指標二：資金流向比重（Turnover Ratio）

**原理：** 以「收盤價 × 成交量」估算各族群的成交額，計算其佔全市場的比重，找出資金最集中的族群。

```
族群 B 今日估算成交額 = 80 億
全市場估算成交額 = 2000 億
→ 資金比重 = 80 ÷ 2000 = 4%（排名第 2）
```

**意義：** 錢往哪裡流，股票就往哪裡漲。資金比重排名前 N 的族群，代表今天市場最積極追逐的方向。

**可調整參數：**
- 資金前幾大族群（預設 5）

#### 集體突破判斷邏輯

| 啟用指標 | 判斷方式 |
|---------|---------|
| 只啟用 HP Density | 創高比例 ≥ 門檻 → 🔥 集體突破 |
| 只啟用 Turnover | 成交額排名 ≤ N → 🔥 集體突破 |
| 兩個都啟用 | 兩個條件同時達到 → 🔥 集體突破 |

### 籌碼面條件：法人買超（可自訂模式）

三大法人條件現在支援兩種模式，在 Sidebar 的「納入三大法人買賣超」區塊操作：

> **注意：** 這個開關只處理外資、投信、自營商買賣超，主要用於投信第一天買超、法人買超加分或額外過濾。v3 / v4 的必要條件「主力連 3 日買超」已改用 Sponsor 分點券商資料，不會因為勾選三大法人而自動取得；請先在「資料管理」補完 Sponsor 主力券商快取。

**模式 A：個別法人皆須買超（嚴格模式）**

可自由勾選外資、投信、自營商，系統會檢查：

- 勾選的法人中，**每一個**都必須在**連續 N 個交易日**為淨買超
- 任何一方賣超，或連續天數不足，都不算達標

這個模式適合你想找「多方資金共識高」的股票。

**模式 B：三大法人合計買超（寬鬆模式）**

這個模式不要求三方各自都買超，而是看整體資金是否淨流入。可選兩種判斷方式：

- **近 N 日累計 > 0**：最近 N 個交易日的三大法人合計淨買超總和大於 0
- **連續 N 日每日 > 0**：最近 N 個交易日中，每一天的三大法人合計淨買超都大於 0

這種模式可減少自營商避險單對訊號的干擾，更適合觀察整體資金方向。

**法人加分規則**

| 模式 | 觸發條件 | 加分 |
|------|---------|------|
| 個別法人皆須買超 | 勾選法人各自連續買超 | +7 |
| 三大法人合計買超 | 合計淨買超達標 | +4 |
| 三大法人合計買超 + 外資投信同步買超 | 合計達標，且外資與投信同時連續買超 | +7（= 4 + 3） |

**必要條件 vs. 加分條件**

- **加分條件（預設）**：符合法人條件才加分，不符合不影響入選
- **必要條件**：勾選「法人條件為必要條件」後，不符合的股票直接剔除

建議：
- 想找「有明確法人背書」的強勢股 → 用「個別法人皆須買超」並設為必要條件
- 想找「整體資金在流入」的股票 → 用「三大法人合計買超」

### 掃描前法人資料確認 Dialog

當你啟用了法人條件（必要條件或加分條件），並且執行**今日掃描**（非歷史回放）時，系統會先確認法人資料的完整程度。若 FinMind 今日尚未充分發布法人資料，頁面會彈出確認對話框。

**確認對話框的三個選擇：**

| 選擇 | 說明 | 適用情境 |
|------|------|---------|
| **排除法人條件** | 忽略所有法人相關條件（含必要條件），以純技術面進行掃描 | FinMind 今日尚未發布，你想先看技術面結果 |
| **用現有資料掃描** | 用目前已有的（可能不完整的）法人資料進行掃描 | 可接受部分資料缺失，想儘早取得初步結果 |
| **取消掃描** | 關閉對話框，不執行掃描 | 想等 FinMind 完整發布後再掃描 |

> **注意：** 歷史回放模式不受此對話框影響，系統會直接排除法人條件並執行掃描（因為歷史日期的法人資料不再更新，完整度已固定）。

> **完整度說明：** FinMind 法人資料並非每日都有全市場 2134 檔；每日實際有三大法人交易的股票通常在 800–1500 檔之間，因此「100% 完整」並沒有固定定義。對話框顯示的筆數僅供參考，用來判斷 FinMind 今日資料是否已大量發布。背景工作器每 5 分鐘持續更新法人快取，若想等完整資料可稍待片刻後再掃描。

### 正確使用選股雷達的步驟

```
第一步：先看「市場環境」頁面，確認大盤現在是多頭還是空頭
第二步：空頭環境 → 選股雷達結果僅供參考，不建議進場
         多頭環境 → 進行以下步驟
第三步：在 Sidebar 設定掃描條件：
         - 流動性過濾：建議選「前日量前100名」
         - 產業輪動：建議勾選「前3個產業」
         - 有法人資料需求時，依風格選擇「個別法人皆須買超」或「三大法人合計買超」
         - 若不想追價過熱股，可開啟「ATR 過熱防護」，預設上限為 MA20 + 3.5 × ATR14
第四步：執行掃描，找出分數 65 分以上的股票
第五步：看「產業族群」頁籤，確認哪個產業最強
第六步：在最強產業中，點入分數最高的個股查看 K 線圖，確認：
         - 趨勢方向清晰嗎？日線、週線都在多頭嗎？
         - 突破盤整了嗎？量能是否集中在上漲日？
         - RS 分數是否 > 70（是否在市場中屬於強勢股）？
第七步：找到合理的進場點和停損位置
第八步：前往「風險控制」計算應該買幾張
第九步：進場後在「交易日誌」新增 BUY 紀錄，系統會自動同步到持股監控
```

## 第九章：持股監控 — 管理手上的股票

### 功能說明

輸入你持有的股票，系統自動計算損益狀態，並在出現 5 種危險訊號時即時發出警示，可推播到 LINE。

### 如何新增持股

進入「管理持股」頁籤，填入以下資料：

| 欄位 | 是否必填 | 說明 |
|------|---------|------|
| 股票代碼 | 必填 | 四位數字，如 2330 |
| 股票名稱 | 選填 | 如台積電，方便識別 |
| 持有股數 | 必填 | 以「股」為單位，例如 1000 股、200 股、7022 股 |
| 成本價 | 必填 | 你的買進均價（元） |
| 停損價 | **強烈建議填寫** | 跌到這個價格執行停損的底線 |
| 停利價 | 選填 | 達到這個價格考慮獲利了結 |
| 備註 | 選填 | 任何想記錄的資訊 |

> **重要更新：** 系統現在以「股數（Shares）」為底層單位，不再以「張數」計算，因此可正確支援零股（例如 1 股、37 股、200 股）。

---

### 持股圖表與布林帶寬

持股監控的個股圖表現在會同時畫出：

- 收盤價走勢
- MA5 / MA10 / MA20
- 布林通道上軌 / 中軌 / 下軌
- 布林通道填色區域
- 布林帶寬百分比

布林帶寬用來衡量目前波動區間有多寬：

```
布林帶寬% = (布林上軌 - 布林下軌) ÷ 布林中軌 × 100%
```

帶寬變窄代表波動收斂，後續可能進入方向選擇；帶寬突然擴大代表波動放大，若同時跌破重要均線或停損價，風險會升高。

---

### 持股資料可直接在網頁編輯

在「管理持股」頁籤中，現在可以直接在表格中批次修改現有資料：

- **不可修改：** 股票代碼、股票名稱
- **可修改：** 股數、成本價、停損價、停利價、備註

這適合以下情境：
- 補登零股加碼後的股數
- 修正匯入後的成本價
- 盤後統一調整停損價、停利價與備註

修改完成後，點擊「儲存目前持股修改」即可一次寫回資料庫。

### 持股資料匯出 / 匯入

「管理持股」頁籤新增了 CSV 匯出與匯入功能。

**匯出功能**

- 點擊「匯出持股 CSV」
- 系統會輸出標準格式欄位：

```
stock_id, stock_name, shares, cost_price, stop_loss, take_profit, notes
```

**匯入功能**

系統支援兩種情境：

1. **系統原生 CSV**：直接讀取前述標準格式
2. **券商匯出檔容錯模式**：可自動辨識常見欄位名稱

欄位對應邏輯：
- 含「股票」或「代碼」→ `stock_id`
- 含「股名」或「名稱」→ `stock_name`
- 含「餘股數」或「股數」→ `shares`
- 含「成本價」→ `cost_price`
- `stop_loss`、`take_profit`、`notes` 若不存在，系統自動補空值

**兩階段確認流程**

1. 上傳 CSV
2. 系統解析後顯示在可編輯表格
3. 你可以先補停損價、停利價、備註
4. 再選擇：
   - **確認並覆寫至資料庫**
   - **確認並附加至資料庫**

> **注意：** 券商 CSV 常見 UTF-8、BIG5、CP950 編碼，系統會自動嘗試判斷。

### 停損價如何設定？

這是最重要的欄位。以下是幾種常見的設定方法：

**方法一：固定比例法（最簡單）**
- 設定成本價的 8-10% 以下為停損
- 例如：成本 100 元 → 停損設在 90-92 元

**方法二：技術支撐位（較精準）**
- 在 K 線圖上找到最近的支撐位（前低、均線、整數關卡）
- 停損設在支撐位下方一點點
- 例如：前低在 95 元 → 停損設在 93 元

**方法三：布林通道下軌**
- 以當前布林通道下軌作為停損參考
- 跌破下軌代表已超出正常波動範圍

> **原則：停損要在進場前就設好，不能在股票跌了之後才想。**

### 五種警示規則完整說明

#### 🔴 危險警示（建議立即檢視，盡快決定）

**危險一：跌破停損價**

- 觸發條件：現價 ≤ 你設定的停損價
- 意義：這是你事先設定的「計畫底線」，跌破代表這筆交易已超出你的風險承受範圍
- 建議行動：依照你的計畫執行停損

停損的數學邏輯：

| 虧損幅度 | 需要的回本漲幅 |
|---------|-------------|
| -5% | +5.3% |
| -10% | +11.1% |
| -20% | +25.0% |
| -30% | +42.9% |
| -50% | +100.0% |

虧損越大，需要的回本幅度呈指數級增加。及早停損是數學上的必然選擇。

**危險二：從近期高點回撤超過 12%**

- 觸發條件：現價比近 60 日最高點跌超過 12%
- 意義：即使沒有設停損，這個幅度代表之前的帳面獲利已大量侵蝕，趨勢可能反轉
- 建議行動：重新評估是否繼續持有，考慮縮減部位

---

#### 🟡 注意警示（需要關注，提前準備應對）

**注意三：從高點回撤 8-12%**

- 觸發條件：回撤幅度在 8% 到 12% 之間
- 意義：開始出現明顯的回撤，尚未到危險程度但需要提高警覺
- 建議行動：收緊停損設定，或考慮先賣出一部分

**注意四：跌破 20 日均線**

- 觸發條件：今日收盤價 < MA20
- 意義：短期趨勢轉弱，買進的理由之一（站上均線）已不再成立
- 建議行動：重新評估持有理由是否仍然存在

**注意五：無停損設定且虧損超過 -5%**

- 觸發條件：沒有填入停損價，且損益% < -5%
- 意義：沒有保護機制的虧損正在擴大
- 建議行動：立刻設定停損價，不要讓虧損繼續無限擴大

---

#### 🔵 提示（好消息，考慮採取行動）

**提示：達到停利目標**

- 觸發條件：現價 ≥ 你設定的停利價
- 意義：計畫中的獲利目標已達到
- 建議行動：考慮全部或部分獲利了結，或上移停損鎖住部分利潤

### Premium 風險警示（Sponsor 試用）

若啟用 FinMind Sponsor / Premium 功能，持股監控會在既有停損、停利、均線、回撤與布林警示之外，額外讀取本機快取中的 Premium 風險訊號。這些警示不會批次呼叫 Premium API，只使用已經存在的 cache。

目前納入的 Premium / 風險來源：

- 官方風險旗標：申報轉讓、注意股、處置股、暫停交易、實施庫藏股。暫停交易會視為最高級警示，因為持倉時可能無法賣出。
- 股權分散籌碼：400 張以上、1000 張以上大戶比例下降，或 10 張以下小股東比例上升。
- 分點主力快取：主力反手或最新主力淨賣超。

這些 Premium 警示會併入原本的持股警示流程，因此畫面警示、警示數、持股表格與 LINE 推播都會一起顯示。畫面上也會有「Premium 風險摘要」區塊，方便你單獨檢查 Premium 來源的警示。「重檢 Premium 風險」按鈕只讀本機 cache，不會重新打 FinMind API，避免每次進持股監控都卡住。

持股表格的「盤中監控」勾選後，還需要在「持股監控 → 即時監控」啟動「盤中即時監控排程」。啟動後，App 內建排程器會在交易日上午 9:00 到 13:30 每分鐘檢查一次，不需要另外開 `python scheduler/jobs.py` 視窗。它會先使用 Sponsor 即時快照 (`taiwan_stock_tick_snapshot`) 取得現價，若快照暫時沒有價格再退回 Sponsor 分 K (`TaiwanStockKBar`)；現價會搭配本機日K計算 MA5 / MA10 / MA20。若現價跌破均線、觸及停損、觸及停利，或在未設定停損時相對成本虧損超過 5%，會推 LINE。相同股票相同條件 60 分鐘內只推一次，避免訊息洗版。

「立即測試一次」會立刻執行一輪監控，不受 13:30 自動排程截止時間限制，方便收盤後排查是否會產生警示。

若「盤中即時監控排程」已啟動，但沒有任何持股勾選「盤中監控」，系統不會呼叫 FinMind 即時快照或分K API，也不會消耗即時資料額度。

### 持股統計指標說明

| 指標 | 計算公式 | 說明 |
|------|---------|------|
| 未實現損益（元） | (現價 - 成本) × 股數 | 目前浮動盈虧的絕對金額 |
| 損益% | (現價 - 成本) ÷ 成本 × 100% | 以成本為基礎的報酬率 |
| 高點回撤% | (現價 - 近60日最高) ÷ 最高 × 100% | 從高點跌了多少，用來評估是否需要保護獲利 |
| MA20 | 20日收盤平均價 | 支撐參考線 |
| 站上MA20 | 現價 > MA20 | 是否仍處於健康趨勢中 |

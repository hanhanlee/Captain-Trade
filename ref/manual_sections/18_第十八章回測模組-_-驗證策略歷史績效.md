## 第十八章：回測模組 — 驗證策略歷史績效

### 功能說明

回測模組讓你用歷史資料**模擬**選股策略的過去表現，驗證策略邏輯是否有效，避免用未經驗證的條件進行實際交易。

> **重要提醒：** 歷史績效不代表未來結果。回測是用來「剔除明顯無效的策略」，而非「保證未來獲利」。

### 開始前：確認資料就緒

回測模組**不提供內建下載按鈕**，所有歷史資料需透過 **6 - 資料管理** 頁面補充：

- 前往「6 - 資料管理」→ 啟動「📼 回測歷史資料重建模式」，系統會在背景自動下載最多 10 年的全市場日K資料
- 回測模組頁面頂端會即時顯示**加權指數（TAIEX）快取狀態**，確認大盤資料正常後再執行

### 兩個步驟

**步驟一：設定參數並執行**（⚙️ 設定回測參數 頁籤）

| 參數 | 說明 | 預設值 |
|------|------|--------|
| 回測期間 | 起始與結束日期 | 最近一年 |
| 使用股票數 | 從本機快取取前 N 檔 | 100 |
| 最低強度分數 | 只買分數達到門檻的訊號 | 65 分 |
| 三線齊穿模式 | 嚴謹型或寬鬆型進場判斷 | 嚴謹型 |
| 個股最大 BIAS | 避免追過熱個股 | 10% |
| 大盤 MA20 濾網 | 大盤條件不佳時暫停進場 | 開啟 |

### 資金管理參數

最近版本的回測不再只是「每次看到訊號就固定買 1 檔」。系統改成更接近真實交易的資金管理模型，會根據帳戶大小、剩餘現金與單筆風險限制來決定是否進場與買多少股。

| 參數 | 說明 | 預設值 |
|------|------|--------|
| 初始資金 | 回測起始帳戶金額，所有績效以此為基準 | 1,000,000 元 |
| 最大持倉檔數 | 同一時間最多持有幾檔股票 | 5 檔 |
| 單筆風險 (%) | 每筆交易最多願意承擔帳戶淨值的幾 % | 2% |
| 買進手續費 | 買進成交成本 | 0.1425% |
| 賣出手續費 | 賣出成交成本 | 0.1425% |
| 證交稅 | 台股賣出交易稅 | 0.3% |

### 系統如何決定買幾股

回測引擎會先估算這筆交易的停損距離，再用「單筆風險預算」反推出可買股數。簡化理解如下：

1. 先算出這筆交易每股最多可能虧多少
2. 用帳戶淨值 × 單筆風險 %，得到這筆交易可承受的最大虧損金額
3. 再同時檢查剩餘現金是否足夠
4. 最後向下取整到台股整張單位（1000 股）

這代表同樣是 100 萬資金，停損比較遠的股票，系統會自動買比較少；停損比較近、風險較小的股票，才有空間買比較多。若同一天有多檔候選標的，系統也會依訊號強度排序，優先把資金分配給分數較高者。

**步驟二：查看績效報告**（📊 查看回測結果 頁籤）
- 勝率、盈虧比、最大回撤、累積報酬等核心指標
- 帳戶淨值曲線圖（以新台幣顯示，含初始資金基準線）
- 出場原因分布圓餅圖
- 每筆交易明細
- 自動診斷建議

### 報告新增了哪些欄位

新版績效報告改成以「帳戶淨值」而不是單純百分比累乘來計算，因此你會看到更多接近實際資金管理的欄位：

| 欄位 | 說明 |
|------|------|
| 初始資金 | 回測一開始投入的本金 |
| 最終淨值 | 回測結束時，現金加上持倉市值的總和 |
| 總損益 | 最終淨值減掉初始資金的絕對金額 |
| 最大回撤（MDD） | 從淨值高點回落的最大幅度 |
| 股數 | 每筆交易實際買進的股數 |
| 成本 / 賣出收入 | 已把手續費與證交稅計入後的真實成本與回收金額 |
| 進場分數 | 該筆交易進場當天的候選強度分數，方便回頭檢查排序邏輯 |

如果你看到某檔股票明明訊號成立卻沒有成交，通常不是 bug，而是以下幾種真實限制之一：

- 當下可用現金不足
- 已達最大持倉檔數
- 按照風險預算換算後，連 1 張都買不起
- 同日有更高分的候選股，資金先被分配掉

### 出場規則：移動式停損/停利

回測引擎使用**移動式**出場規則，比傳統固定停損更能讓利潤奔跑：

#### 移動停損（預設 8%）

傳統固定停損從「買入價」計算，一旦設定就不動。

移動停損改為從**持倉期間最高價**計算：
```
停損觸發價 = 持倉最高價 × (1 - 停損幅度%)

例：買入 100 元，股票漲到 120 元
  → 停損線從「92 元」移到「110.4 元」（120 × 0.92）
  → 後來跌回 110.4 → 自動出場，保住 10.4 元的利潤
```

**優點：** 股票繼續漲時，停損線跟著上移，保護已累積的帳面獲利。

#### 移動停利（預設啟動門檻 15%，啟動後追蹤 5%）

當股票達到啟動門檻後，追蹤幅度自動縮緊，更積極鎖住利潤：

```
未達 +15%：使用移動停損（8% 追蹤）
達到 +15% 後：切換為移動停利（5% 追蹤）

例：買入 100 元，漲到 120 元（+20%，已超過 15% 啟動門檻）
  → 移動停利線 = 120 × 0.95 = 114 元
  → 後來跌回 114 → 出場，鎖住 14 元的利潤
```

#### 其他出場條件

| 條件 | 說明 |
|------|------|
| 跌破 MA5 / MA10 / MA20 | 可自行選擇哪條均線作為跌破出場條件（可關閉） |
| 持滿 N 天 | 最長持有天數限制，到期強制出場（預設 20 天） |

#### 出場原因分析

| 出場原因 | 意義 |
|---------|------|
| 🔥 移動停損 | 從最高點下跌超過停損幅度出場（保護性出場） |
| ✅ 移動停利 | 達到啟動門檻後緊縮追蹤，鎖住利潤出場 |
| 跌破MA5 / MA10 / MA20 | 依你在回測頁設定的均線週期提前離場 |
| 持滿N天 | 超時強制出場 |

### 績效指標解讀

| 指標 | 健康範圍 | 說明 |
|------|---------|------|
| 勝率 | ≥ 40% | 台股短線策略正常範圍 |
| 盈虧比 | ≥ 1.5 | 平均獲利是平均虧損的 1.5 倍以上 |
| 最大回撤 | > -25% 為佳 | 策略運行期間的最大虧損幅度 |
| 移動停損比例 | < 50% | 超過一半以停損出場代表訊號品質偏低 |
| 移動停利比例 | > 40% 理想 | 代表策略在目標內緊縮鎖利的能力良好 |

### 回測的常見誤解

**誤解一：「回測績效 80%，未來也會有 80%」**
回測是用歷史資料優化的，存在「過擬合」風險。建議保守解讀，用來排除明顯差的策略，而非預測確切的未來績效。

**誤解二：「回測沒有交易成本，不準確」**
本系統的回測**已內建台股交易成本**：
- 買進手續費：0.1425%
- 賣出手續費：0.1425%
- 交易稅（賣出）：0.3%
- 合計約 0.447%，已接近實際環境

**誤解三：「用越多年的資料越準確」**
資料越舊，市場結構可能已改變（例如 2018 年以前的台股結構和現在差異很大）。建議重點看最近 2-3 年的表現，參考 5-10 年的趨勢。

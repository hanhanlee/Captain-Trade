## 第二十一章：問題回報 — 回報問題與功能建議

### 功能說明

「問題回報」頁面用來收集使用者遇到的錯誤、操作疑問與功能建議。送出後，系統會把內容存成 Markdown 檔，放在專案根目錄的 `user_reports/` 資料夾，方便後續集中整理與處理。

### 問題回報

選擇「問題回報」後，需要填寫：

- 發生問題的頁面，可選「其他」
- 複製步驟 / 問題描述
- 預期結果
- 實際結果

### 功能新增

選擇「功能新增」後，需要填寫：

- 想新增功能的頁面，可選「其他」
- 功能描述

### 檔案保存位置

每次送出會建立一個獨立 Markdown 檔，檔名包含時間、回報類型與頁面名稱，例如：

```text
user_reports/20260420_142500_bug_2-持股監控_a1b2c3.md
```

`user_reports/.gitkeep` 只用來保留資料夾；實際使用者回報的 `.md` 檔不會被 Git 追蹤，避免把使用者描述或敏感內容誤提交。

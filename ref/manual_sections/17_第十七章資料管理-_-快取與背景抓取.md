## 第十七章：資料管理 — 快取與背景抓取

### 為什麼需要本機快取？

FinMind 免費帳號每小時只有 600 次 API 請求。全市場約 950 檔普通股，每次掃描若都從網路抓取，很快就會超出限制。

系統的解法是：**把歷史股價存在你的電腦裡**（本機快取），掃描時先讀本機資料，只有遇到新資料才呼叫 API。

### 背景預抓取工作器

App 啟動後，系統會自動在背景開始下載全市場的歷史價格資料。你不需要做任何事，它會靜靜地跑在背景。

| 時段 | 每小時上限 | 說明 |
|------|----------|------|
| 交易時間（09:00–15:05）| 100 次 | 保留 500 次給手動操作 |
| 非交易時間 | 500 次 | 全力填充快取 |
| 全速重建模式 | 600 次 | 開放全部額度，手動啟動 |

### 休市模式

連假或停市期間，法人資料不會更新。開啟「🏖️ 休市模式」可讓法人快取永不過期，避免反覆呼叫 API 浪費配額。

- **開啟位置：** 「6 - 資料管理」頁面最上方的切換按鈕
- **效果：** 法人資料快取無限期有效，不消耗 API 額度
- **恢復交易後：** 手動關閉此開關，下次掃描時系統會自動更新法人資料

### 快取品質診斷怎麼看

「🔍 快取品質診斷」會把本機資料分成 5 類，幫你快速找出為什麼某些股票掃不到、回測跑不動、或資料明顯不完整：

| 分類 | 意義 | 你應該怎麼做 |
|------|------|-------------|
| 完全缺失 | 股票清單裡有這檔，但本機完全沒有任何價格快取 | 用「手動補抓」或啟動重建模式 |
| 資料不足 | 有資料，但天數低於你設定門檻（例如少於 100 天） | 若要做掃描或回測，建議補齊 |
| 資料過舊 | 最新一筆資料離今天太久，可能停留在數天前 | 先刷新快取，再重新掃描 |
| 已下市 | 該股票在 `ref/suspendList.csv` 中，系統視為已下市或停止交易 | 不列入問題股票，不需要反覆補抓 |
| 全部快取摘要 | 顯示所有股票的最早/最新日期與天數分布 | 用來整體判斷資料庫健康度 |

這個診斷的目的不是把所有股票都補到一模一樣，而是讓你先把「真的會影響使用」的問題資料找出來。系統會把已下市股票排除在主要問題清單之外，避免你把配額浪費在不需要修復的代碼上。

### 市場指數資料（TAIEX）為什麼重要

回測模組現在會使用 **TAIEX（台灣加權股價指數）** 當作市場基準。這筆資料主要用在：

- 比較策略績效是否優於大盤
- 支援相對強弱（RS）或市場基準相關的計算
- 避免基準曲線空白，或因市場資料缺失導致報告不完整

如果資料管理頁看到「📈 市場指數下載（TAIEX）」尚未完成，建議先補齊，再執行正式回測。否則即使個股資料完整，部分報表或比較基準仍可能缺值。

**回測歷史重建的狀態顯示**

啟動「📼 回測歷史資料重建」後，頁面上方會直接顯示：

- 目前是否處於回測重建模式
- 回測待補股票數
- 正在抓取哪一檔（會顯示 `[回測] 股票代碼`）
- 回測歷史重建進度條
- 最近一次抓取時間

若遇到 429 限額，頁面也會顯示暫停倒數，可按「⚡ 立即恢復」提前繼續。

### 背景工作器的優先順序

系統背景工作器在填充快取時，依以下優先順序自動執行，不需手動介入：

```
1. 回測歷史重建模式（手動啟動）— 全速抓取 10 年深度歷史
2. 一般價格快取（自動）— 補齊 400 天近期價格
3. 基本面快取（自動）— 價格快取全部補齊後，自動抓取財報指標（90天TTL）
```

### 首次安裝建議流程

1. 啟動 App（`streamlit run app.py`）
2. 前往「6 - 資料管理」
3. 確認工作器狀態為「🟢 運行中」
4. 睡前點擊「🔨 重建資料庫」→ 雙重確認 → 全速重建（補近 400 天）
5. 隔天起床後，大部分股票已有 400 天歷史資料
6. 若需要回測使用：睡前點擊「📼 回測歷史資料重建」→ 補充最多 10 年資料（約 950 次 API）
7. 之後每天背景自動維持快取，掃描幾乎零 API 消耗

### 備援機制（yfinance）

當 FinMind 達到每小時 600 次上限時，系統自動切換至 Yahoo Finance 備援：
- 頁面顯示**橘色警告橫幅**
- 股價資料仍可取得（可能有 15 分鐘延遲）
- **三大法人條件自動停用**（Yahoo Finance 無此資料）
- 下一個小時額度重置後自動恢復 FinMind

## 第二十四章：資料備份與雲端同步

> 「資料是比程式碼更難重建的東西。」

### 為什麼備份很重要

`srock.db` 是本工具的唯一資料儲存，包含：
- 所有持股紀錄與成本
- 完整的交易日誌與績效統計
- 選股快取與市場資料
- 系統設定與排程狀態

**一旦硬碟損壞或意外刪除，上述資料全部消失，無法從程式碼復原。**
自動備份腳本確保每天都有一份安全副本存在 Google Drive，且最多只損失一天的資料。

---

### 備份腳本位置

```
scripts/backup_gdrive.py
```

---

### 執行流程

| 步驟 | 動作 | 技術細節 |
|------|------|---------|
| 1 | 安全複製 DB | `sqlite3.Connection.backup()`，WAL 模式相容，不鎖表 |
| 2 | gzip 壓縮 | level 6，串流壓縮，不整塊讀入記憶體 |
| 3 | 上傳至 gdrive: | `rclone copyto`，以 list 格式傳遞參數（路徑含空格安全）|
| 4 | 驗證上傳 | `rclone ls` 確認檔案確實存在於雲端後才刪本機暫存 |
| 5 | 刪除本機暫存 | `temp_backup.db` 和 `temp_backup.db.gz` 全部清除 |
| 6 | 清除舊備份 | `rclone delete --min-age 14d --include srock_backup_*.db.gz` |

---

### 首次部署設定

#### 步驟一：確認 Rclone 已安裝

```bash
rclone version
```

若未安裝，至 https://rclone.org/downloads/ 下載並加入 PATH。

#### 步驟二：服務帳號金鑰（重要）

本工具使用 Google Cloud 服務帳號授權 Rclone 操作 Google Drive。
金鑰檔案位於：

```
E:\workspace\srock tool\secrets\gcp_backup.json
```

> ⚠️ **此檔案已列入 `.gitignore`（`secrets/` 目錄整體忽略），絕對不會被提交到 Git。**
> 若遺失此檔案，請至 GCP Console → IAM → 服務帳號 → 新增金鑰（JSON 格式），
> 重新下載後放回相同路徑，**不需重新設定 Rclone**。

#### 步驟三：確認 Rclone Remote 設定

```bash
rclone listremotes
# 應顯示 gdrive:
```

若未設定，執行：

```bash
rclone config
# 新增 remote → 名稱設為 gdrive → 選 Google Drive
# 若使用服務帳號：service_account_file = 填入 gcp_backup.json 的完整路徑
```

#### 步驟四：測試連線

```bash
rclone ls gdrive:
# 應顯示雲端資料夾內容（或空列表）
```

### .env 設定

備份腳本的錯誤與成功通知透過 Telegram 推播，需在 `.env` 設定：

```env
TELEGRAM_BOT_TOKEN=your_bot_token_here

# 三選一，優先序由上至下：
TELEGRAM_CHAT_ID=           # 建議：備份專用通知頻道
TELEGRAM_SYSTEM_CHAT_ID=    # 已有系統頻道時使用
TELEGRAM_STOCK_CHAT_ID=     # 最後備援
```

### 手動執行備份

```bash
cd "E:\workspace\srock tool"
python scripts/backup_gdrive.py
```

正常完成後，Telegram 會收到：
```
[srock 備份成功] 2026-04-27 16:30
srock_backup_20260427.db.gz 已上傳至 gdrive:
（14 天前舊備份已自動清除）
```

任何步驟失敗，Telegram 會收到完整錯誤訊息（traceback），便於排查。

### 自動排程（Windows 工作排程器）

建議於每日收盤後自動執行：

1. 開啟「工作排程器」→「建立基本工作」
2. 觸發程式：每日 **16:30**（收盤後，確保 DB 寫入完成）
3. 動作：
   - 程式：`python`
   - 引數：`scripts/backup_gdrive.py`
   - 起始位置：`E:\workspace\srock tool`

---

### 備份檔命名規則

```
srock_backup_YYYYMMDD.db.gz
```

每日一份，保留 **14 天**（超過自動刪除）。5 TB 空間即使每份 100 MB 也能存放多年，不需手動管理。

### 緊急還原步驟

若需要從備份還原：

```bash
# 1. 列出可用備份
rclone ls gdrive: --include "srock_backup_*.db.gz"

# 2. 下載目標備份
rclone copyto "gdrive:srock_backup_20260427.db.gz" temp_restore.db.gz

# 3. 解壓縮
python -c "
import gzip, shutil
with gzip.open('temp_restore.db.gz', 'rb') as f_in, open('srock_restore.db', 'wb') as f_out:
    shutil.copyfileobj(f_in, f_out)
"

# 4. 備份現有 DB（保險）後替換
move srock.db srock.db.bak
move srock_restore.db srock.db

# 5. 清理
del temp_restore.db.gz
```

### 常見問題

| 問題 | 原因 | 解決方式 |
|------|------|---------|
| `rclone: command not found` | Rclone 未安裝或未加 PATH | 重新安裝 Rclone 並確認 PATH 設定 |
| `gdrive: not found` | Remote 未設定 | 執行 `rclone config` 新增 gdrive remote |
| `permission denied` | 服務帳號金鑰失效 | GCP Console 重新生成金鑰並取代 `secrets/gcp_backup.json` |
| 上傳驗證失敗 | 網路不穩或 Google Drive API 限流 | 等待幾分鐘後手動重跑；Telegram 已收到錯誤通知 |
| `TELEGRAM_BOT_TOKEN 未設定` | .env 未設定 | 填寫 `.env` 中的 Telegram 設定 |
